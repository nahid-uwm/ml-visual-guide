🎲 Naive Bayes & Probability

Questions 1–10
1
Multiple Choice
Given that using a joint probability distribution one can compute any probability over the random variables, then why not always obtain the joint probability distribution for any task?
  • Conditional probabilities cannot be estimated from joint probability distributions.
  • Most probabilities are zeros in joint probability distributions.
  • For most tasks no joint probability distribution can be defined.
  • Often an exponential number of parameters need to be estimated to obtain a joint probability distribution.
Why correct: A joint distribution over n binary features needs 2ⁿ−1 parameters — exponential. Infeasible to estimate from finite data.
Others wrong: Conditionals CAN be computed from joints (just sum and divide). Joints aren't zero-heavy by default. A joint can always be defined for any set of random variables — just impractical.
2
Multiple Choice
If A and B are features and C is the target, with P(A|C) = 0.2 and P(B|C) = 0.3, what is P(A|B,C) under the naïve Bayes assumption?
  • 0.5
  • 0.3
  • 0.2
  • 0.06
Why correct: Naïve Bayes assumes features are conditionally independent given the target. So P(A|B,C) = P(A|C) = 0.2 — knowing B adds no info once C is fixed.
Others wrong: 0.06 is P(A|C)·P(B|C) = the joint, not the conditional. 0.3 is P(B|C). 0.5 is just a random average.
3
True / False
Naïve Bayes model computes the joint probability distribution of the features and the target.
  • True
  • False
Why correct: NB models P(Y) and P(Xᵢ|Y) only — never the full joint. Whole point is to AVOID estimating the joint, which would need exponential parameters.
4
Multiple Choice
Naïve Bayes requires estimating a number of probabilities that is:
  • Quadratic in terms of the number of features
  • Independent of the number of features
  • Exponential in terms of the number of features
  • Linear in terms of the number of features
Why correct: One P(Xᵢ|Y) per feature × per class value, plus P(Y). For n features and k classes: roughly n·k probabilities → linear in n.
Others wrong: Exponential is the joint distribution we're trying to avoid. Quadratic would need feature-pair probabilities. Probability count clearly grows with n, so not independent.
5
Multiple Choice
From the training data — 5 examples with label positive/negative, colors red/red/red/green/blue, labels positive/positive/negative/negative/positive — what is the estimated P(red | positive)? 
Ex 1: small  red    circle    positive
Ex 2: large  red    circle    positive
Ex 3: small  red    triangle  negative
Ex 4: medium green  square    negative
Ex 5: large  blue   circle    positive
  • 1
  • 3/5
  • 2/5
  • 2/3
Why correct: Out of 3 positive examples (Ex 1, 2, 5), 2 of them are red (Ex 1 and 2). So P(red | positive) = 2/3.
Others wrong: 3/5 is overall fraction of red examples — that's P(red), not conditional. 2/5 is positive-AND-red over total — that's joint P(red, positive). 1 would mean every positive is red — Ex 5 is blue, so no.
6
Multiple Choice
What is the naïve Bayes assumption?
  • Features are independent.
  • Target is conditionally independent of each feature given the other features.
  • Features are independent of the target.
  • Features are conditionally independent of each other given the target.
Why correct: The "naïve" part is exactly this — once you know the target Y, features become independent of each other. That lets P(X₁..Xₙ|Y) factor into the product Π P(Xᵢ|Y).
Others wrong: Plain independence (no conditioning) is too strong and usually false. The opposite (target independent of features) destroys the model. NB doesn't claim features are independent of the target — that would make the target useless.
7
Multiple Choice
Given features X₁, X₂, …, Xₙ and target Y, a machine learning task can be formulated as the probabilistic task of estimating:
  • P(X₁)·P(X₂)·…·P(Xₙ)·P(Y)
  • P(X₁, X₂, …, Xₙ | Y)
  • P(X₁, X₂, …, Xₙ, Y)
  • P(Y | X₁, X₂, …, Xₙ)
Why correct: At inference you SEE the features and want to predict Y. That's the posterior P(Y|features).
Others wrong: P(X|Y) is the likelihood (used inside NB but not the goal). The joint P(X,Y) is what NB avoids. The product form assumes total independence — that's not the predictive question.
8
Multiple Choice
What probabilities are needed to compute P(Y|X₁,…,Xₙ) using the naïve Bayes method?
  • P(Xᵢ|Y) for all i.
  • P(Y) and P(Xᵢ|Y) for all i.
  • P(Y|X₁,…,Xₙ) and P(Y)
  • P(Y) and P(Xᵢ) for all i.
Why correct: Bayes: P(Y|X) ∝ P(Y)·Π P(Xᵢ|Y). Need both the prior P(Y) AND each class-conditional P(Xᵢ|Y).
Others wrong: P(Xᵢ|Y) alone misses the prior. P(Y|X..) is the answer we're trying to compute — circular. P(Xᵢ) doesn't capture how features depend on the class.
9
Multiple Choice
By Bayes' theorem, P(disease | symptom) =
  • P(symptom) · P(disease)
  • P(symptom | disease) · P(disease) / P(symptom)
  • P(symptom | disease) · P(symptom) / P(disease)
  • P(symptom | disease)
Why correct: Bayes' theorem: P(A|B) = P(B|A)·P(A) / P(B). Plug in A=disease, B=symptom.
Others wrong: P(symptom)·P(disease) assumes independence and ignores the conditional. Swapping numerator and denominator gives the wrong direction. P(symptom|disease) alone is just the likelihood — missing prior and normalizer.
10
Multiple Answer (select all true)
Which statements about Bayes' theorem are true?
  • It is an approximation.
  • It is based on the naïve Bayes assumption.
  • It lets us compute P(X|Y) in terms of P(Y|X).
  • It can be derived from the definition of conditional probability.
Why correct: Bayes' theorem is an exact identity derived from P(A,B) = P(A|B)·P(B) = P(B|A)·P(A). It's not an approximation. It flips conditional direction — that IS its purpose. The naïve Bayes assumption is a SEPARATE simplification ON TOP of Bayes' theorem.
Others wrong: Bayes' theorem is exact, not approximate. It exists independently of NB — Bayesian statistics uses it everywhere without the NB assumption.

📍 kNN, Distances & Feature Selection

Questions 11–20
11
Multiple Choice
k-Nearest Neighbors can be extended for regression by:
  • Predicting the majority class
  • Using cosine similarity
  • Predicting the average target values of the k nearest neighbors
  • Normalizing the features
Why correct: Regression needs a numeric output. Average (or weighted average) of neighbors' targets is the natural extension.
Others wrong: Majority class is for classification. Cosine similarity changes the distance metric, not the prediction type. Normalization is a preprocessing trick, not what makes kNN do regression.
12
Multiple Choice
As the number of dimensions increases (curse of dimensionality):
  • Sampling density increases.
  • k-nearest neighbor method becomes more accurate
  • An example is less likely to be near another example.
  • Fewer examples are needed to maintain the same sampling density.
Why correct: In high dimensions, volume grows exponentially → points become roughly equidistant → the concept of "near" weakens. kNN's nearest neighbor stops being meaningfully closer than far ones.
Others wrong: Sampling density DECREASES (volume grows). kNN gets WORSE in high dim. You need MORE examples (exponentially more) for the same density.
13
Multiple Choice
How does Dynamic Time Warping (DTW) overcome the disadvantages of Euclidean distance for time series?
  • By first best aligning the two time series.
  • By first mapping the two time series to high dimensional features.
  • By first scaling the two time series to be of the same length.
  • By first normalizing the two time series.
Why correct: Euclidean compares point-by-point at the same index, which fails when one series is shifted or stretched. DTW finds the best non-linear alignment (warping) between the two before computing distance.
Others wrong: DTW doesn't lift to higher dim or normalize — its trick is purely the warped alignment. It handles series of DIFFERENT lengths, so it doesn't force them equal.
14
Multiple Choice
Data normalization is needed to:
  • Find the best value of parameters.
  • Prevent some features from numerically dominating other features.
  • Prevent overfitting.
  • Remove outliers.
Why correct: If one feature is in thousands and another in 0–1 range, distance/gradient calculations get dominated by the large one. Normalization (scale to similar range or zero-mean unit-variance) prevents this.
Others wrong: Normalization doesn't directly find parameters (training does). It doesn't prevent overfitting (regularization does). It doesn't remove outliers — outliers are still outliers, just on a smaller scale.
15
Multiple Answer (select all true)
Which types of features should be retained after feature selection?
  • Redundant
  • Predictive
  • Interacting
  • Irrelevant
Why correct: Predictive features carry signal about the target — obviously keep. Interacting features may look weak alone but combine to predict well (XOR pattern) — keep them too.
Others wrong: Redundant features duplicate information already in others — wasted dimensions, drop them. Irrelevant features add noise to distance/likelihood — drop.
16
Multiple Choice
Which of the following measures is based on the ratio between feature values?
  • Cosine similarity
  • Edit distance
  • Dynamic time warping distance
  • Euclidean distance
Why correct: Cosine similarity = (a·b)/(‖a‖·‖b‖) — depends on the angle, not magnitudes. Scale a vector by 2 and cosine stays the same → it's about the RATIO of feature values, not their absolute size.
Others wrong: Edit distance counts string-edit operations. DTW is alignment-based. Euclidean is √Σ(aᵢ−bᵢ)² — absolute difference, not ratio.
17
Multiple Choice
Forward sequential feature selection could:
  • Miss including interacting features.
  • Include redundant features.
  • Include irrelevant features.
  • Miss including predictive features.
Why correct: Forward selection adds features one at a time based on individual performance. Two features that are weak alone but powerful together (XOR-like) never get picked, because adding either one alone doesn't help.
Others wrong: Redundant and irrelevant features won't improve performance alone, so forward selection won't pick them. It WILL pick predictive features (that's what it's designed for).
18
Multiple Answer (select all true)
If there are 100 training examples, which are true among kNN models with different k?
  • 100-nearest neighbor will be most prone for overfitting.
  • 100-nearest neighbor will be most prone for underfitting.
  • 1-nearest neighbor will be most prone to overfitting.
  • 1-nearest neighbor will be most prone to underfitting.
Why correct: k=1 means decision boundary fits noise exactly — overfitting. k=n=100 means EVERY query averages all training data → predicts majority class always → severe underfitting.
Others wrong: The roles are reversed: large k smooths and underfits; small k is jittery and overfits.
19
Multiple Choice
Two examples (binary features): 
Example 1 — F1: True,  F2: True,  F3: False
Example 2 — F1: True,  F2: False, F3: False
What is the Jaccard similarity?
  • 2/3
  • 0
  • 1/3
  • 1/2
Why correct: Jaccard = (number of features True in BOTH) / (number of features True in AT LEAST ONE). True-True features: just F1 → 1. True-in-at-least-one: F1, F2 → 2. So 1/2. (F3 is False-False, ignored.)
Others wrong: 2/3 would treat False-False as a match (it isn't, in Jaccard). 1/3 would count F3 in the union. 0 would require zero overlap, but F1 matches.
20
Multiple Choice
What computation is done during training in k-nearest neighbor method?
  • Nothing
  • Computing distances between every pair of training examples
  • Finding the nearest training example to each training example
  • Computing the Voronoi diagram
Why correct: kNN is a "lazy learner". Training = just store the data. All distance computation happens at test time when a query arrives.
Others wrong: Pairwise distances and Voronoi diagrams are sometimes precomputed as optimizations, but they're not REQUIRED by the algorithm. The standard definition does no training work.

📊 Evaluation Metrics

Questions 21–30
21
Multiple Choice
If a dataset has 1000 examples, which has the largest number of training examples per fold?
  • 2-fold cross-validation
  • 1000-fold cross-validation
  • 100-fold cross-validation
  • 10-fold cross-validation
Why correct: In k-fold CV, training set per fold = n − n/k. With k=1000 (leave-one-out): train on 999, test on 1. With k=2: train on 500. Bigger k → bigger training set per fold.
Others wrong: Smaller k means a larger test slice and a smaller training set per fold. So 2-fold and 10-fold both train on fewer examples than 1000-fold.
22
Multiple Choice
A model predicted 90 test examples to be positive and was correct for 80 of them. What is 80/90?
  • Recall
  • Specificity
  • Sensitivity
  • Precision
Why correct: Precision = TP / (TP + FP) = "of those predicted positive, how many are actually positive". 80/90 fits this exactly.
Others wrong: Recall/Sensitivity = TP/(TP+FN) — over all REAL positives, not predicted. Specificity = TN/(TN+FP) — about negatives.
23
Multiple Choice
Which evaluation measure depends on the class distribution?
  • Recall
  • Sensitivity
  • Specificity
  • Precision
Why correct: Precision's denominator (TP+FP) depends on how many negatives get misclassified — i.e., depends on how many negatives there are. Change the negative-to-positive ratio and precision shifts even if the classifier is unchanged.
Others wrong: Sensitivity (TP/(TP+FN)) depends only on positives. Specificity (TN/(TN+FP)) depends only on negatives. Recall = sensitivity. None care about the OVERALL distribution between classes.
24
Multiple Choice
Different points on an ROC curve are obtained using:
  • Different random baselines.
  • Different thresholds on the classification confidence.
  • Different sizes of the test data.
  • Different folds of cross-validation.
Why correct: An ROC curve sweeps the classification threshold. Each threshold gives one (FPR, TPR) point. Lower threshold → predict more positives → higher TPR and FPR.
Others wrong: ROC is computed on a fixed test set. Baselines and CV folds don't change shape — they evaluate different runs but each gives ONE point per threshold.
25
Multiple Choice
Out of all positives in the data, how many were correctly predicted as positive — this is:
  • Specificity
  • Sensitivity or Recall
  • Accuracy
  • Precision
Why correct: By definition, Sensitivity = Recall = TP / (TP + FN) = "of real positives, fraction caught".
Others wrong: Specificity is the analog for negatives. Accuracy mixes both. Precision flips the question — "of predicted positives, how many real".
26
True / False
Accuracy cannot be 100% when some non-zero numbers in a confusion matrix are not on the diagonal.
  • True
  • False
Why correct: Off-diagonal cells are misclassifications. If ANY of them is non-zero, you have at least one mistake → accuracy < 100%.
27
Multiple Choice
Parameters of a machine learning method are tuned on which dataset?
  • Test set
  • Entire data
  • Validation set
  • Training set
Why correct: Hyperparameters and model selection use the validation set. The training set fits the model; the test set is touched only once at the end.
Others wrong: Tuning on test data leaks information — your reported test accuracy becomes overestimated. Tuning on the training set is what training does (for parameters, not hyperparameters). The entire dataset is what you split FROM.
28
Multiple Choice
Location on an ROC curve of a model whose predictions are all correct?
  • Bottom-left corner
  • Top-left corner
  • Bottom-right corner
  • Top-right corner
Why correct: ROC x-axis is FPR (False Positive Rate, lower = better), y-axis is TPR (True Positive Rate, higher = better). Perfect prediction → FPR=0, TPR=1 → top-LEFT.
Others wrong: Bottom-left = no positives predicted at all (TPR=0). Top-right = predict everything positive (TPR=1 but FPR=1). Bottom-right = worst possible (all flipped).
29
True / False
In machine learning evaluation there should always be some overlap between training and test data.
  • True
  • False
Why correct: Overlap = data leakage. Test set must be UNSEEN by the model during training so the reported performance reflects generalization to new data.
30
Multiple Choice
AUC of a model is 0.7. What can be concluded?
  • Its specificity is 0.3
  • It is predicting better than a random prediction model.
  • Its sensitivity is 0.7.
  • It cannot achieve sensitivity higher than 0.7.
Why correct: Random guessing gives AUC = 0.5. Anything above that beats random. 0.7 is moderate but better than chance.
Others wrong: AUC is a single summary number — it doesn't fix sensitivity or specificity. You can pick a threshold to get sensitivity=0.9 even if AUC=0.7 (at the cost of higher FPR).

🌳 Decision Trees

Questions 31–40
31
Multiple Choice
Which is TRUE about decision trees?
  • They are human-interpretable
  • They have no inductive bias
  • They do not overfit
  • They cannot be rewritten as rules
Why correct: A trained tree is a chain of if-then rules a human can read top-to-bottom. That's exactly why they're popular in medicine, law, etc.
Others wrong: Every ML method has inductive bias (DT's is "prefer smaller trees"). DTs overfit very easily (deep trees memorize). DTs literally ARE rewritable as rules — each root-to-leaf path is one rule.
32
True / False
Decision trees are prone to overfitting because their representation language is very expressive.
  • True
  • False
Why correct: A deep enough tree can carve out a leaf for every training example, fitting noise perfectly. Solution: prune, or limit max_depth.
33
Multiple Choice
Decision trees can be extended to do regression by:
  • Binning the numeric target into categories
  • Decision trees cannot be extended for regression
  • Reduced error pruning
  • Using variance instead of entropy for choosing the best feature to split with
Why correct: For continuous targets, replace entropy/Gini (categorical) with variance reduction (numeric). At each leaf, predict the mean target value of training examples in that leaf.
Others wrong: Binning loses information and isn't true regression. DT clearly CAN do regression. Reduced error pruning is a separate technique (anti-overfitting), not what enables regression.
34
Multiple Choice
Three non-leaf nodes A, B, C in a tree being pruned by Reduced Error Pruning. Validation accuracy on full tree: 90%. After pruning A: 80%. After pruning B: 95%. After pruning C: 92%. What does the method do next?
  • Stop pruning
  • Prune B
  • Prune A
  • Prune C
Why correct: REP prunes the node whose removal IMPROVES validation accuracy the most. Pruning B gives 95% (best gain). Then continue with the new tree.
Others wrong: Pruning A drops accuracy (don't do it). Pruning C improves by less than B. REP doesn't stop unless NO pruning helps — and pruning B does help.
35
Multiple Choice
Decision trees handle numeric features by:
  • Using variance instead of entropy for choosing the best feature to split with
  • Decision trees cannot be extended to work with numeric features.
  • Using numeric threshold to split.
  • Reduced error pruning
Why correct: For a numeric feature x, find best threshold t (often midpoint between sorted values) and split into x ≤ t vs x > t. Information gain is computed for each candidate threshold.
Others wrong: Variance vs entropy is for regression TARGETS, not numeric FEATURES. DTs CAN handle numeric features (this is the whole answer). REP is unrelated.
36
Multiple Choice
Which of these has the highest entropy?
  • {A, A, A, B, B, B}
  • {A, A, A, A, A, A}
  • {A, A, B, B, B, B}
  • {A, B, B, B, B, B}
Why correct: Entropy is maximized at 50/50. {3A, 3B} is exactly 50/50 → entropy = 1 bit. Any imbalance reduces entropy.
Others wrong: {6A, 0B} → entropy = 0 (perfectly pure). {2A, 4B} → entropy ≈ 0.92. {1A, 5B} → entropy ≈ 0.65. None beat 1.0.
37
Multiple Choice
ID3 has preference bias toward:
  • Smaller trees
  • Broader trees
  • Balanced trees
  • Larger trees
Why correct: ID3's greedy information-gain selection favors features that split data into purer subsets quickly → shorter paths from root to leaf → smaller trees. This is the "Occam's razor" inductive bias built into the algorithm.
Others wrong: ID3 doesn't deliberately balance the tree or make it broad. Larger trees would only emerge if pruning is disabled and noise is fitted — that's a side effect, not a bias.
38
Multiple Choice
A regression tree can be distinguished from a classification tree by looking at its:
  • Leaves
  • Branches
  • Internal nodes
  • Root
Why correct: Classification leaves contain class labels. Regression leaves contain numeric values (typically the mean of training targets falling there).
Others wrong: Internal nodes test features (same in both types). Branches are just feature-value transitions. The root is also a feature test.
39
Multiple Choice
Four decision trees with (train, test) accuracies: 
A: 99% / 80%
B: 95% / 94%
C: 70% / 60%
D: 80% / 85%
Which is overfitting?
  • A
  • C
  • B
  • D
Why correct: Overfitting = high train, low test (big gap). Tree A: 99 vs 80 → 19-point gap. Memorized training data, failed to generalize.
Others wrong: B: tiny gap, well-fit. C: both low → underfitting. D: test > train means it's not even fitting training well — neither overfit nor underfit in the classic sense.
40
True / False
Decision tree induction algorithm is recursive.
  • True
  • False
Why correct: The algorithm: pick best split, partition data into subsets, then recursively build a tree on each subset. Each recursive call returns a subtree.

🎓 ML Foundations

Questions 41–50
41
Multiple Choice
What manual work is needed to apply machine learning?
  • Change targets to suit the features.
  • Create training examples.
  • Create rules relating features to targets.
  • Mathematically derive targets from features.
Why correct: You must provide labelled training examples. That's the supervisory signal — the rest (rules, parameter values) is learned automatically.
Others wrong: If you could derive targets mathematically or write rules, you wouldn't need ML. Changing targets to fit features is backwards.
42
True / False
Unbiased learning leads to the best prediction performance.
  • True
  • False
Why correct: An unbiased learner has no preference among hypotheses consistent with the training data — it cannot generalize beyond the training set. Inductive bias (a preference) is REQUIRED to generalize.
43
Multiple Choice
Which is NOT a suitable task for machine learning?
  • Recognize fingerprints.
  • Predict when Jupiter and Saturn will be nearest to each other.
  • Predict tomorrow's stock market index.
  • Diagnose a patient's disease.
Why correct: Planetary positions are governed by exact physics (Kepler's laws, Newtonian gravity). Use math/simulation, not ML. ML is for problems where the underlying function is unknown or too complex to express analytically.
Others wrong: Fingerprints, diseases, and stock markets have noisy, complex, hard-to-formalize patterns → exactly where ML shines.
44
Multiple Choice
How many possible models are there for a task with 6 binary features and 5 possible target values?
  • 2⁶ × 5
  • 5 × 6
  • 2⁶ + 5
  • 5^(2⁶) = 5^64
Why correct: A "model" is a function from feature combinations to a target. There are 2⁶ = 64 possible feature combinations. Each can independently map to any of 5 targets. So 5 choices per combination, raised to 64 combinations → 5^64. Astronomically large — proves you need inductive bias to navigate this hypothesis space.
Others wrong: Multiplying or adding feature count to target count misses that you're counting FUNCTIONS, not points. The right operation is exponentiation.
45
Multiple Choice
Overfitting will take place:
  • Independent of the representation function (language bias).
  • If the representation function (language bias) is more expressive than needed.
  • If there is no underfitting.
  • If the representation function (language bias) is not expressive enough.
Why correct: Overfitting happens when the model can express patterns too fine-grained for the true signal → it ends up fitting noise. Excess capacity vs the task is the cause.
Others wrong: Insufficient expressiveness causes UNDERFITTING, the opposite. The presence/absence of overfitting depends very much on the representation, not independent of it. "No underfitting" doesn't imply overfitting — could be just right.
46
True / False
Training data provides sufficient rationale to make predictions on unseen data.
  • True
  • False
Why correct: Training data alone is consistent with many hypotheses (infinitely many functions can fit a finite dataset). Without inductive bias (Occam's razor, smoothness assumptions, etc.), we cannot prefer one extrapolation over another.
47
Multiple Choice
What makes two machine learning methods different from each other?
  • They use different features.
  • They have different inductive biases.
  • They are applied on different tasks.
  • They are trained on different training datasets.
Why correct: Two algorithms can use the same features and data but produce different models because they prefer different hypothesis classes (linear vs tree vs neural). The inductive bias IS what makes each algorithm distinct.
Others wrong: Features, datasets, and tasks are inputs to a method — they're not what makes the method itself different.
48
Multiple Choice
Machine learning is an ill-posed problem because:
  • Often multiple models are consistent with the training data.
  • The goal of machine learning is often not clear.
  • Often no model is consistent with the training data.
  • Machine learning task is often not well-defined.
Why correct: "Ill-posed" means the solution isn't unique. Many hypotheses fit any finite training set equally well — choosing among them requires assumptions (inductive bias).
Others wrong: ML goals (predict target Y) ARE clear. Failing to find ANY consistent model is a separate (different) problem. Task definition isn't the issue here.
49
Multiple Choice
What is predicted in a machine learning task is called:
  • Feature value
  • Target
  • Feature
  • Training data
Why correct: Target (also "label", "response variable", "output Y") is what the model predicts. Features are the inputs.
50
Multiple Choice
For machine learning to work:
  • Training and test data should come from the same distribution.
  • Training and test data should be the same.
  • Training data should be larger than test data.
  • Training and test data should come from different distributions.
Why correct: The i.i.d. assumption — train and test draw from the same distribution. Otherwise patterns learned won't apply (distribution shift → bad generalization).
Others wrong: Same data = no evaluation. Larger train is a practical heuristic, not a strict requirement. Different distributions break the foundation of supervised learning.
