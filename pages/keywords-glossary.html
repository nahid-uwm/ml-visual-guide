Neural Networks & Training

Perceptron → MLP → backprop → activations → regularization.
High-level: Stack neurons in layers. Each neuron computes weighted sum + bias, applies non-linear activation. Train by gradient descent + backpropagation. Use ReLU for hidden layers, softmax/sigmoid for output, cross-entropy loss for classification, dropout + early stopping to prevent overfitting.
TermMeaning (easy)Why / Where UsedExam Likelihood
PerceptronSingle neuron with step activation. Learns linear decision boundary.Update rule w ← w + η(y − ŷ)x. Convergence theorem.HIGH
Learning rate (η)Step size for weight updates. Too big = diverge; too small = slow.Every gradient descent update.HIGH
Weight (w)How much each input contributes to neuron's output.The learnable parameter. What training adjusts.HIGH
Bias (b)Constant shift added to weighted sum. Lets neuron fire even with zero input.Always +b in z = w·x + b. Updated like weights.HIGH
Activation functionNon-linear function applied after weighted sum. Without it, stacking layers = same as one layer.ReLU/sigmoid/tanh in hidden; softmax/sigmoid in output.HIGH
Sigmoid σ(z)Squashes any real number to (0, 1). Derivative ≤ 0.25.Binary output. Avoid in deep hidden layers (vanishing gradient).HIGH
ReLU max(0, z)Outputs z if positive, else 0. Derivative = 1 when active.Default for hidden layers in deep nets. Solves vanishing gradient.HIGH
Leaky ReLUSmall slope (0.01) for negative inputs instead of zero. Prevents dead neurons.When dying ReLU problem appears in training.HIGH
SoftmaxConverts logits to probabilities summing to 1. Biggest logit → biggest probability.Output layer for multi-class classification.HIGH
One-hot encodingBinary vector with 1 at correct class index, 0 elsewhere.Target representation for multi-class classification.HIGH
BackpropagationCompute gradients layer-by-layer using chain rule, output → input.Every neural net training step.HIGH
Chain ruledL/dw = dL/dŷ · dŷ/dz · dz/dw. Multiply local derivatives.Foundation of backprop.HIGH
Cross-entropy lossL = −log(p_true). Big penalty for confident-wrong prediction.Classification loss. Pair with softmax/sigmoid output.HIGH
MSE lossL = ½(y − ŷ)². Squared error.Regression. Wrong choice for classification.MED
Gradient descentMove weights opposite to gradient direction to reduce loss.Standard training algorithm.HIGH
Vanishing gradientGradient shrinks toward zero in deep networks. Early layers stop learning.Diagnosing why deep sigmoid nets fail. Fix: ReLU, BN, skip connections.HIGH
Dying ReLUNeuron stuck at zero output forever because input always negative.Common failure. Fix: Leaky ReLU, smaller LR, better init.MED
DropoutRandomly zero some neurons during training. Forces robustness.Regularization. Reduces overfitting. Off at test time.HIGH
Early stoppingStop training when validation error stops improving.Free regularization. Save best-validation checkpoint.HIGH
Universal approximationTheorem: 1 hidden layer with enough neurons approximates any continuous function.Why MLPs are powerful in theory.MED
XOR problemTwo classes arranged so no straight line separates them.Historical reason perceptron failed → MLPs invented.MED
🎯 Exam hot spots Perceptron update calculation (very common). ReLU vs sigmoid choice + why (very common). Cross-entropy vs MSE for classification. Compute σ'(z) using ŷ(1−ŷ) trick.

Convolutional Neural Networks

Filters slide across image with shared weights. Detect local patterns anywhere.
High-level: Designed for grid-like input (images). Conv layer applies small learned filters across the image. Pooling shrinks size + adds invariance. Stack conv+pool+conv+pool, then FC head. Transfer learning + augmentation key for small datasets.
TermMeaning (easy)Why / Where UsedExam Likelihood
Convolution layerSlides small filter across input, computes weighted sum at each position.Detect local patterns. Replace dense layer for images.HIGH
Filter / KernelSmall matrix of learnable weights (e.g., 3×3, 5×5).Each filter detects one type of feature (edges, textures, etc.).HIGH
Feature mapOutput of one filter applied across the input.N filters → N feature maps as output.HIGH
Stride (S)How many pixels the filter jumps between applications.S=2 halves output size; S=1 keeps it nearly full.HIGH
Padding (P)Extra zero pixels added around input borders."Same" padding preserves size; "valid" (P=0) shrinks.HIGH
Output size formula⌊(W − F + 2P) / S⌋ + 1.Must know cold. Tested in every CNN exam.HIGH
Parameter count(F·F·C_in + 1) × C_out per conv layer.Tested. Don't forget bias and input channels.HIGH
Weight sharingSame filter weights reused at every position.Why CNNs have far fewer params than dense layers.HIGH
Translation equivarianceShift input → feature map shifts by same amount.Why CNNs detect features anywhere in image.HIGH
Translation invarianceShift input → output unchanged.Achieved through pooling. Different from equivariance.MED
Max poolingOutput max of each small window (e.g., 2×2).Reduce size, add invariance. No learnable parameters.HIGH
Average poolingOutput average of window instead of max.Alternative to max pooling. Smoother.MED
Receptive fieldRegion of input that affects one output unit. Grows with depth.Deeper layers "see" larger image patches.MED
Channels (C_in, C_out)Depth of input/output. RGB image has C_in=3.Each filter spans all input channels.HIGH
Transfer learningTake pretrained CNN, replace last layer, fine-tune on new task.Small dataset scenarios. Avoids overfitting.HIGH
Fine-tuningRe-train last few layers (or all) on new task data.Adapt pretrained model. Use small learning rate.MED
Data augmentationRandom flips, rotations, crops applied to training images.Multiply data + add robustness. Free regularization.MED
U-NetEncoder-decoder CNN with skip connections.Image segmentation. Per-pixel labels.LOW
YOLOSingle-pass CNN that predicts bounding boxes + classes.Real-time object detection.LOW
🎯 Exam hot spots Conv output size (5×5 image, 3×3 filter → 3×3 output). Param count for given filter+channels. Conv vs dense advantages. Max pooling output of small region.

RNN / LSTM / GRU

Networks with memory. Process sequences one step at a time.
High-level: Same weights reused at every timestep. Hidden state carries memory forward. Plain RNN suffers vanishing gradient over long sequences. LSTM/GRU add gates and cell state to remember long-range info. Trained by BPTT (Backprop Through Time).
TermMeaning (easy)Why / Where UsedExam Likelihood
RNNNetwork that processes one token at a time, passing hidden state forward.Sequential inputs: text, time series, audio.HIGH
Hidden state h_tVector summarizing all past inputs up to time t.The RNN's "memory". Updated at every step.HIGH
Weight sharing (time)Same W_hx, W_hh, W_yh at every timestep.Why param count is independent of sequence length.HIGH
W_hx, W_hh, W_yhInput→hidden, hidden→hidden, hidden→output weight matrices.Three matrices define a basic RNN.HIGH
UnrollingDrawing the RNN expanded across time. Same network repeated.Visualize BPTT path. Identify which connections carry gradient.HIGH
BPTTBackprop applied to the unrolled RNN. Sum gradients across time.RNN training.HIGH
Vanishing gradient (RNN)Gradient shrinks at each step back → early steps don't learn.Why plain RNNs fail on long sequences. Fix: LSTM.HIGH
Exploding gradientGradient grows huge during BPTT → unstable training.Fix: gradient clipping (cap norm).MED
LSTMRNN variant with cell state + 3 gates (forget, input, output).Long-range dependencies. Solves vanishing gradient.MED
Cell state C_tLong-term memory conveyor belt that flows through LSTM unchanged unless gates modify it.Why LSTM remembers long-range info.MED
Forget gateSigmoid that decides what to erase from cell state.Drop irrelevant past info.MED
Input gateSigmoid that decides what new info to write into cell state.Add new memory.MED
Output gateSigmoid that decides what part of cell state to expose as hidden state.Control what passes forward.MED
GRUSimpler LSTM variant with 2 gates instead of 3.Faster, similar performance.LOW
🎯 Exam hot spots Unrolled RNN diagram: identify W_hx, W_hh, W_yh on each connection. Backprop path for Y_t (which connections carry gradient). Why vanishing gradient happens. LSTM mitigation.

Attention & Transformers

Each token looks at every other token. Foundation of all modern LLMs.
High-level: Each input token emits Query (asks question), Key (advertises), Value (provides info). Query·Key gives attention weight (softmaxed). Output is weighted sum of values. Multi-head runs many in parallel. Replaces RNNs because fully parallel + handles long-range.
TermMeaning (easy)Why / Where UsedExam Likelihood
Query (Q)"What am I looking for?" — projection of current token.Compared against all keys to get attention weights.HIGH
Key (K)"What do I contain?" — projection of each token (what each advertises).Match against query via dot product.HIGH
Value (V)"What I pass forward" — actual content returned if selected.Weighted sum gives output for the query.HIGH
Attention formulasoftmax(QKᵀ / √d_k) · V.Core mechanism. Memorize.HIGH
d_k scaling (√d_k)Divide QKᵀ by √d_k before softmax. Prevents huge scores from saturating softmax.Numerical stability of softmax gradients.MED
Self-attentionQ, K, V all come from same sequence. Each token attends to every other in the same input.Encoder layers. Context-dependent word embeddings.HIGH
Encoder-decoder attentionQ from decoder, K and V from encoder.Lets decoder look at encoded input while generating output. Translation.HIGH
Masked self-attentionCausal mask blocks each token from attending to future tokens.Decoder. GPT-style generation (predict next token).MED
Multi-head attentionRun h attention heads in parallel, each in a d_k = d_model/h subspace.Different heads learn different relationships (syntax, semantics, position).HIGH
Positional encodingVector added to input embeddings to inject word order info.Attention has no built-in order awareness. PE fixes that.MED
TransformerArchitecture using only attention + feed-forward layers (no recurrence).All modern NLP. Foundation of BERT, GPT.MED
Word embeddingDense vector representation of a word (100–300 dims).Replace one-hot. Captures semantic similarity.MED
BERTEncoder-only transformer pretrained with masked language modeling.Understanding tasks: classification, QA.LOW
GPTDecoder-only transformer trained on next-token prediction.Generation tasks: chat, completion.LOW
🎯 Exam hot spots Identify Q/K/V from a given equation (e.g., "ate' = f(I)·g(h(ate),i(I)) + ..."). Encoder-decoder attention: where do Q vs K/V come from. Why attention beats RNN: parallelism + long-range.

Generative AI: GAN, VAE, Diffusion, Autoencoder

Models that produce new examples instead of classifying.
High-level: Autoencoder = compress-and-reconstruct (representation learning). GAN = generator vs discriminator, adversarial training. VAE = autoencoder with structured latent space. Diffusion = denoise from noise step-by-step, very stable training.
TermMeaning (easy)Why / Where UsedExam Likelihood
AutoencoderEncoder shrinks input to bottleneck; decoder reconstructs. Trained on itself.Representation learning, anomaly detection, denoising.HIGH
Bottleneck layerNarrow middle layer that forces compression.Learns compact features. Cannot copy directly through.HIGH
EncoderNetwork that compresses input to a smaller representation.First half of autoencoder/VAE.HIGH
DecoderNetwork that expands compressed representation back to full input.Second half of autoencoder/VAE.HIGH
GANTwo networks: Generator makes fakes, Discriminator detects them. Trained adversarially.Generate realistic novel images.HIGH
Generator (G)Takes random noise vector → outputs fake sample.The part you keep after training. Generates new samples.HIGH
Discriminator (D)Classifies images as real (training data) or fake (from G).Provides gradient signal to train G. Discarded at inference.HIGH
Random vector (z)Latent noise input to generator. Usually ~N(0, I).Different z → different generated image.HIGH
Real vs Fake labelsReal images labeled 1, generated images labeled 0 (for D).Discriminator's training labels.HIGH
Minimax lossmin_G max_D V(D, G). G minimizes; D maximizes.Adversarial game.MED
Mode collapseGenerator outputs only one (or few) type of sample. Loses diversity.Common GAN failure mode.MED
VAEAutoencoder where encoder outputs distribution params (μ, σ) instead of point.Smooth latent space. Sampleable.LOW
Diffusion modelAdd noise to image step-by-step (forward), train net to reverse one step.Image generation (Stable Diffusion, DALL-E).LOW
Conditional GANFeed class label as extra input to G and D.Control what gets generated (e.g., "generate a cat").LOW
🎯 Exam hot spots GAN component identification (A=random vector, B=Generator, C=Discriminator, D=Training data). Which is trained / kept after training. Autoencoder bottleneck purpose. Why autoencoders for representation learning.

Reinforcement Learning

Learn by trial-and-error from rewards. No labeled correct actions.
High-level: Agent observes state, picks action, gets reward, lands in next state. Goal: learn policy that maximizes total reward over time. Q-learning: estimate Q(s, a) = expected future reward. Pick action with highest Q. Bellman update refines estimates iteratively.
TermMeaning (easy)Why / Where UsedExam Likelihood
State (s)Current situation the agent observes.Input to policy and Q function.HIGH
Action (a)Choice the agent makes from available options.What the agent does in state s.HIGH
Reward (R)Scalar feedback after taking an action.Only learning signal (no labeled correct actions).HIGH
Next state (s')State agent lands in after taking action a from s.Used in Bellman update.HIGH
Policy π(a|s)Strategy: which action to take in each state.What the agent learns. Optimal policy = pick argmax Q.HIGH
Q-value Q(s, a)Expected total future reward if you take action a in state s.Core estimate in Q-learning.HIGH
Discount γWeight on future rewards. 0 = myopic, 1 = patient.Always between 0 and 1. Typical: 0.9 or 0.99.HIGH
Learning rate αStep size for Q updates. Different from γ.Larger α = faster but noisier. Smaller = stable but slow.HIGH
Bellman updateQ(s,a) ← (1−α)·Q(s,a) + α·[R + γ·max Q(s', a')].Core Q-learning equation.HIGH
TD targetR + γ·max Q(s', a'). The "what Q should be" estimate.Drag old Q value toward this.MED
TD errorTD target − current Q. How wrong the estimate was.Magnitude of update direction.MED
ε-greedyWith prob ε pick random action (explore); otherwise pick argmax Q (exploit).Balance exploration vs exploitation.MED
DQNQ-learning with deep network approximating Q instead of table.Large/continuous state spaces (e.g., Atari from pixels).LOW
Q-tableLookup table storing Q(s, a) for every state-action pair.Tabular Q-learning for small discrete problems.MED
🎯 Exam hot spots Define Q-value in one sentence. Identify s, a, s', α, R, Q in Bellman equation. Compute new Q given current Q, R, γ, α. Pick action with highest Q.

Unsupervised Learning & Clustering

Find structure without labels. Clustering, PCA, autoencoders.
High-level: No labels available. K-Means partitions into K centroid clusters. HAC builds hierarchical tree. DBSCAN groups by density. PCA reduces dimensions by projecting onto max-variance axes.
TermMeaning (easy)Why / Where UsedExam Likelihood
K-MeansPick K. Iterate: assign each point to nearest centroid, then update centroid to cluster mean.Fast partitioning into K spherical clusters.HIGH
Centroid (μ)Mean position of points in a cluster.K-Means moves these to minimize within-cluster distance.HIGH
HACHierarchical Agglomerative Clustering. Start with every point alone, merge closest pair repeatedly.Build dendrogram. No K required upfront.HIGH
DendrogramBinary tree showing merge order and merge heights.Cut at any height → that many clusters.MED
Single-linkDistance between clusters = MIN distance between any two members.Captures chains and elongated clusters.HIGH
Complete-linkDistance between clusters = MAX distance between any two members.Compact, spherical clusters.MED
Average-linkDistance between clusters = mean of all pairwise distances.Balanced choice between single and complete.MED
Ward linkageMerge pair that minimizes within-cluster variance increase.Like K-Means objective but hierarchical.LOW
Elbow methodPlot inertia vs K, find bend ("elbow") = good K.Choose K visually.MED
Silhouette scores = (b − a) / max(a, b). +1 well clustered, 0 ambiguous, −1 wrong cluster.Evaluate clustering quality without labels.MED
K-Means++Smart init: spread first centroids far apart.Avoids bad random starts.LOW
DBSCANDensity-based clustering. Groups points with enough neighbors within ε.Arbitrary cluster shapes. Finds noise points.LOW
PCAProject data onto directions of maximum variance (eigenvectors of covariance).Dimensionality reduction, decorrelation.MED
Eigenvalue (λ)Variance along principal component direction.Sort descending → top components first.MED
Explained variance ratioλᵢ / Σλⱼ. Fraction of total variance captured by component i.Choose k for 90% or 95% retention.MED
🎯 Exam hot spots HAC iterations on small 1D dataset using single-link. K-Means iteration trace. Why K-Means fails on concentric circles. Pick K via elbow/silhouette.

Theory: SVM, Naïve Bayes, Trees, kNN, Bias-Variance

Classical methods + fundamental concepts.
High-level: Linear → polynomial → logistic → SVM chain. SVM picks max-margin hyperplane. Naïve Bayes assumes feature independence given target. kNN votes among k closest training points. Decision trees split on information gain. Bias-variance trade-off explains under/overfitting.
TermMeaning (easy)Why / Where UsedExam Likelihood
Linear regressionŷ = w·x + b. Predict continuous output.Regression baseline. Closed-form solution.HIGH
Logistic regressionLinear regression squashed through sigmoid → probability.Binary classification with linear boundary.HIGH
SVMFind hyperplane that maximizes margin between classes.Strong generalization. Pick "middle" separator.HIGH
Margin (SVM)Distance from hyperplane to nearest training point on either side.SVM maximizes this for best generalization.HIGH
Support vectorTraining point on the margin boundary.Only support vectors determine the hyperplane.MED
Kernel trickReplace x·x' with K(x, x') to operate in high-dim space implicitly.Non-linear boundaries without explicit features.MED
Naïve BayesP(Y|X) ∝ P(Y)·∏ P(Xᵢ|Y). Assumes feature independence given Y.Fast text classification. Counts only.HIGH
NB assumptionFeatures are conditionally independent given the target.What makes it "naïve". Often violated but works ok.HIGH
Laplace smoothingAdd small count to avoid zero probabilities in NB.When P(X|Y) = 0 would kill the whole product.MED
Bayes theoremP(A|B) = P(B|A) · P(A) / P(B).Flip conditional probabilities. NB foundation.HIGH
kNNFind k closest training points to query. Predict majority (classification) or mean (regression).Non-parametric. No training. Slow at test time.HIGH
k (in kNN)Number of nearest neighbors to consult.k=1 overfits, k=n underfits (predicts majority class).HIGH
Curse of dimensionalityIn high dimensions, all points are roughly equidistant. Distance loses meaning.Why kNN degrades for d > 20. Use PCA first.HIGH
Decision treeRecursively split data on most informative feature.Human-readable rules. Handles mixed features.HIGH
EntropyH = −Σ pᵢ log pᵢ. Measures uncertainty. 0 = pure, max = balanced.Decision tree split criterion.HIGH
Information gainEntropy before split − weighted entropy after. Pick max-gain split.ID3 / decision tree feature selection.HIGH
PruningRemove branches that don't improve validation accuracy.Prevent decision tree overfitting.HIGH
BiasSystematic error from too-simple model (underfitting).Linear model on curved data → high bias.HIGH
VarianceSensitivity to training data noise (overfitting).Deep tree, 1-NN → high variance.HIGH
Bias-variance tradeoffReducing one usually increases the other. Total error = bias² + variance + noise.Diagnose under/overfit. Tune model complexity.HIGH
OverfittingLow train error, high test error. Model memorized noise.Symptom of high variance.HIGH
UnderfittingBoth train and test error high. Model too simple.Symptom of high bias.HIGH
Inductive biasAssumptions a learner uses to generalize beyond training data.Different inductive biases = different generalizations from same data.HIGH
One-hot encoding (features)Replace nominal feature with binary indicator per value.Convert categorical to numeric for algorithms that need it.HIGH
L1 regularizationAdd λ·Σ|wⱼ| to loss. Produces sparse weights.Feature selection effect.LOW
L2 regularizationAdd λ·Σwⱼ² to loss. Shrinks all weights smoothly.General regularization. No sparsity.LOW
🎯 Exam hot spots kNN majority vote calculation (1-NN vs 3-NN vs k=n). NB probability estimation from data table. SVM separator identification (max margin = middle). Information gain calculation. One-hot encoding example. Bias vs variance diagnosis.

Evaluation Metrics & Validation

How to measure model performance correctly.
High-level: Confusion matrix → TP, FN, FP, TN. From these derive sensitivity, specificity, precision, recall, accuracy, F1. ROC curve plots sensitivity vs (1−specificity) at different thresholds. AUC summarizes ROC in one number. Cross-validation gives robust estimates.
TermMeaning (easy)Why / Where UsedExam Likelihood
Confusion matrix2×2 table: TP, FP, FN, TN counts.Source of all classification metrics.HIGH
TP / FN / FP / TNTrue positive / False negative / False positive / True negative.Building blocks of every metric.HIGH
SensitivityTP / (TP + FN). Of all real positives, how many caught.Same as recall. Critical when missing positives is bad (medical).HIGH
SpecificityTN / (TN + FP). Of all real negatives, how many correctly left alone.Critical when false alarms are costly.HIGH
PrecisionTP / (TP + FP). Of all predicted positives, how many really were.When false positives matter (spam, alerts).HIGH
RecallSame as sensitivity. TP / (TP + FN).Memorize: Recall = Sensitivity.HIGH
Accuracy(TP + TN) / Total. Fraction of all predictions correct.Misleading on imbalanced data.HIGH
F1 scoreHarmonic mean of precision and recall = 2PR/(P+R).Single number balancing both. Use for imbalanced data.MED
ROC curvePlot sensitivity (y) vs 1−specificity (x) at different thresholds.Shows full sensitivity-specificity tradeoff.HIGH
AUCArea Under ROC Curve. 1 = perfect, 0.5 = random.Summarize ROC in one number.HIGH
Top-left corner (ROC)(0, 1) point = perfect classifier (100% sensitivity, 100% specificity).ROC curve closest to top-left = best.MED
Training setData used to fit model parameters.Learn weights here.HIGH
Validation setData used to tune hyperparameters and pick best model.Choose K, learning rate, etc. Never tune on test set.HIGH
Test setHeld-out data for final evaluation. Touch only once.Estimate generalization. Must stay unseen until done.HIGH
k-fold cross-validationSplit data into k folds. Train on k−1, test on 1. Rotate. Average results.Robust performance estimate. Standard practice.HIGH
LOOCVLeave-one-out cross-validation. k = n. Train on n−1, test on 1.Max training data per fold. Slow.MED
Learning curvePlot of accuracy vs training set size.Diagnose: more data helping or plateaued?HIGH
Class imbalanceOne class much more frequent than another.Accuracy misleading. Use F1, AUC, balanced accuracy.MED
🎯 Exam hot spots Compute sensitivity, specificity, precision, recall, accuracy from a 60+/40− split. Recall = sensitivity (always trips people up). Cross-validation fold construction. ROC top-left = perfect. Why not tune on test set.

Exam-Day Strategy

Scan-and-recall checklist for the last hour.
What to study last: Anything marked HIGH. These are the questions most likely on the exam.

Memorize cold:
  • CNN output size: ⌊(W − F + 2P)/S⌋ + 1.
  • CNN params: (F·F·C_in + 1) × C_out.
  • Perceptron update: w ← w + η(y − ŷ)x.
  • Bellman: Q ← (1−α)Q + α[R + γ·max Q'].
  • Sigmoid derivative: σ'(z) = ŷ(1 − ŷ).
  • Attention: softmax(QKᵀ/√d_k) · V.
  • NB: P(Y|X) ∝ P(Y)·∏ P(Xᵢ|Y). Need P(Y) + each P(Xᵢ|Y).
  • Sensitivity = Recall = TP/(TP+FN).
  • Precision = TP/(TP+FP).
Recognize-the-method clues:
  • "Reward, action" → Reinforcement Learning.
  • "Image classification, edges anywhere" → CNN.
  • "Generate realistic novel images" → GAN.
  • "Reconstruct input from compressed" → Autoencoder.
  • "Sequence of words" → RNN/LSTM/Transformer.
  • "Context depends on other words" → Attention.
  • "No labels, find groups" → Clustering (K-Means, HAC).
  • "Counts probabilities, fast training" → Naïve Bayes.
  • "Closest training point" → kNN.
  • "Max margin separator" → SVM.
  • "If-then rules, interpretable" → Decision Tree.
