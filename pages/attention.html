Attention & Transformers

Self-attention, Query/Key/Value, transformer architecture, and Large Language Models.

Part 1

Word Representations

How do we represent words as input to a neural network?

Why Numerical Vectors?

The best way to represent a word for a neural network is using a numerical vector (embedding). Why?

❌ One-hot encoding: Sparse, no semantic meaning, arbitrary ordinal relationships.

❌ Single integer: Imposes arbitrary ordinal relationships (cat=2, dog=3 doesn't mean dog > cat).

✅ Dense embeddings (Word2Vec, learned): Capture semantic relationships in continuous vector space. Work well with gradient-based learning.

Semantic Geometry

Embeddings encode meaning in geometry:

Classic example
\[\text{king} - \text{man} + \text{woman} \approx \text{queen}\]

Similar words cluster together in embedding space. Analogical reasoning emerges naturally from the vector arithmetic.

Transformers learn their own embeddings end-to-end from the data — often better than pre-trained word vectors for the specific task.

Part 2

Self-Attention

Every word attends to every other word simultaneously — unlike RNNs which process sequentially.

The Core Idea

For each word (Query), compute how much it should "attend to" every other word (Keys), then take a weighted sum of Values.

Attention
\[\text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V\]
Query
Q
What I'm looking for
Key
K
What I can offer
Value
V
What I actually share

In self-attention: Q, K, and V are all computed from the same input sequence (via linear projections W_Q, W_K, W_V). This is what makes it "self"-attention — each position attends to all other positions within the same sequence.

Adaptation: The Query, Key, and Value projection matrices (W_Q, W_K, W_V) are learnable parameters. During training or fine-tuning, these matrices adapt the attention computation to the specific task.

Self-Attention Heatmap

Click a word (query) to see which words it attends to most. Brighter = higher attention weight.

Attention Score Computation — Worked Example

Sentence: "The sky is blue" — computing attention for query word "is"

Step 1: Dot Products

dp(is, sky) = 0.3
dp(is, is) = 0.9
dp(is, blue)= 0.6

Step 2: Softmax → Weights

w(sky) = 0.17
w(is) = 0.51
w(blue)= 0.32
Sum = 1.0 ✓

Step 3: Weighted Sum of Values

new_rep("is") =
0.17·v(sky)
+ 0.51·v(is)
+ 0.32·v(blue)

Quiz: Which is the Query?

In "sky·dp(is,sky) + is·dp(is,is) + blue·dp(is,blue)" — the query is "is".

The query is the word whose new representation is being computed.

Part 3

Transformer Architecture

Encoder vs Decoder

ComponentAttention TypeUsed For
EncoderSelf-attention onlyUnderstanding input
DecoderMasked self-attn + encoder-decoder attnGenerating output

Encoder-decoder attention (cross-attention): Q comes from the decoder's own sequence, K and V come from the encoder's output. This is what uses "encoder-decoder attention" in the transformer architecture — the Decoder.

Masked self-attention: In the decoder, future tokens are masked — prevents the model from "cheating" by looking at tokens it hasn't generated yet.

Positional Encoding

Self-attention has no inherent notion of word order. Positional encodings add information about each token's position:

Sinusoidal PE
\[\text{PE}_{(pos,2i)} = \sin\!\left(\frac{pos}{10000^{2i/d}}\right)\]

Added to token embeddings before the first layer. Modern models use learned positional embeddings.

Part 4

Large Language Models (LLMs)

How LLMs are Trained

Large language models are pre-trained on the next word prediction task. LLMs like GPT are trained autoregressively: given all previous tokens, predict the next one.

This self-supervised objective over massive text corpora (trillions of tokens) produces rich, general-purpose language representations.

1
Pre-training
Train on massive text corpus with next-token prediction. Learns language structure, facts, reasoning.
2
Fine-tuning / RLHF
Adapt to specific tasks or align with human preferences using labeled data or human feedback.
3
Few-shot / Zero-shot
At inference, provide examples in the prompt (few-shot) or just instructions (zero-shot).

Self-Supervised Learning

In self-supervised learning, correct targets are automatically generated from the data itself — e.g., masking a word and predicting it, or predicting the next word — without requiring manual human annotation.

BERT vs GPT

ModelArchitectureTask
BERTEncoder onlyUnderstanding (classification, QA)
GPTDecoder onlyGeneration (text completion)
T5/BARTEncoder-DecoderTranslation, summarization

Sentiment/emotion classification (happy, sad, angry, neutral) is a language understanding task — only the encoder (e.g. BERT) is needed. The decoder (used for text generation) is not needed here.

Few-Shot Learning

Giving large language models a few illustrative examples of a task in the prompt is called few-shot learning (or in-context few-shot learning). No gradient updates to the model — examples just guide it in the prompt.
