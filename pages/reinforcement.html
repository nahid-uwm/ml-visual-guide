Markov Decision Process (MDP)

RL problems are modeled as MDPs: the future depends only on the current state (Markov property).

MDP Tuple
\[\mathcal{M} = (S, A, P, R, \gamma)\]
S = states, A = actions, P = transition probabilities,
R = reward function, γ = discount factor
Discounted Return
\[G_t = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \cdots = \sum_{k=0}^{\infty} \gamma^k r_{t+k}\]

Types of RL Algorithms

Value-Based

Learn Q(s,a) or V(s). Extract policy by choosing action with highest Q. Examples: Q-learning, DQN.

Policy-Based

Directly learn π(a|s). Optimize policy gradient. Examples: REINFORCE, PPO.

Model-Based

Learn model of environment, use it to plan. Examples: Dyna-Q, AlphaZero.

Actor-Critic

Combine value and policy: Actor selects actions, Critic evaluates them. Examples: A3C, SAC.

Part 2

Q-Learning

Learn the optimal action-value function Q*(s,a) using the Bellman equation — without needing a model of the environment.

Bellman Equation

Optimal Q-function
\[Q^*(s,a) = r + \gamma \max_{a'} Q^*(s', a')\]

The value of taking action a in state s = immediate reward r + discounted best future value.

Q-Learning Update Rule

TD Update
\[Q(s,a) \leftarrow Q(s,a) + \alpha\!\left[r + \gamma\max_{a'}Q(s',a') - Q(s,a)\right]\]

α = learning rate, γ = discount factor

TD error = \(r + \gamma\max_{a'}Q(s',a') - Q(s,a)\) — the difference between target and current Q value. Drive this toward zero.

Exploration vs Exploitation

ε-Greedy Policy
\[\pi(s) = \begin{cases} \text{random action} & \text{with prob. } \varepsilon \\ \arg\max_a Q(s,a) & \text{with prob. } 1-\varepsilon \end{cases}\]

Start with high ε (explore), decay over time (exploit more). Balances trying new actions vs. using what you know.

Part 3

Deep Reinforcement Learning

Deep Q-Network (DQN)

Use a neural network to approximate Q(s,a). Takes raw pixels as state, outputs Q-value for each action.

Key tricks: Experience replay (random sampling from memory), target network (stable learning target), reward clipping.

AlphaGo, Atari games

Policy Gradient

Directly optimize the policy π(a|s;θ) using gradient ascent on expected reward.

REINFORCE
\[\nabla_\theta J = \mathbb{E}[G_t \nabla_\theta \log \pi_\theta(a_t|s_t)]\]
PPO, TRPO

RL Challenges

⏱️ Credit assignment — which action caused the reward?
🔍 Exploration — how to find new rewarding strategies?
🌊 Non-stationarity — policy changes the data distribution
📊 Sample efficiency — millions of steps needed
🎯 Reward shaping — designing good reward functions is hard
