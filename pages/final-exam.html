🧠 Neural Networks & Perceptron Backprop · Activation · Loss
1
A perceptron has weights w = [0.5, −0.3] and bias b = 0.1. Input x = [1, 2], true label y = 1, learning rate η = 0.2. Compute the output, determine if an update is needed, and show the new weights.
⌄
The Equation
Symbol Breakdown
SymbolMeaningIn this problem
wWeight vector — how much each input matters[0.5, −0.3]
xInput feature vector[1, 2]
bBias — shifts the decision boundary0.1
ŷPredicted output (0 or 1)computed below
yTrue label1
ηLearning rate — step size for updates0.2
y − ŷError signal (drives the update)1 − 0 = 1
Step-by-Step Solution
  1. Compute net input: z = w·x + b = (0.5)(1) + (−0.3)(2) + 0.1 = 0.5 − 0.6 + 0.1 = 0.0
  2. Apply step function: ŷ = 1 if z > 0 else 0 → ŷ = 0 (z = 0, not strictly > 0)
  3. Error = y − ŷ = 1 − 0 = 1 → update needed
  4. New w₁ = 0.5 + 0.2 · 1 · 1 = 0.7
  5. New w₂ = −0.3 + 0.2 · 1 · 2 = 0.1
  6. New b = 0.1 + 0.2 · 1 = 0.3
Plain English
The perceptron is like a judge who votes Yes/No. It adds up weighted evidence (inputs × weights) and flips on if it passes a threshold. When it's wrong, it shifts its weights toward the correct answer — heavier toward inputs that should've pushed it over.
When Used / When Not
✅ Binary linearly-separable data. Historically: spam vs. not-spam on simple features.
❌ XOR problem, multi-class, non-linear boundaries — use MLP instead.

2
Explain backpropagation. A network has output ŷ = 0.8, true y = 1, uses MSE loss and sigmoid activation. Compute the loss and the gradient flowing back from the output neuron.
⌄
Key Equations
Symbol Breakdown
SymbolMeaningValue here
LLoss — how wrong the prediction is0.02
ŷNetwork's predicted output0.8
yGround-truth label1
σ(z)Sigmoid function output0.8
σ'(z)Sigmoid derivative = σ(z)(1−σ(z))0.16
∂L/∂wGradient — direction & magnitude to adjust weightcomputed below
Step-by-Step
  1. Loss: L = ½(y − ŷ)² = ½(1 − 0.8)² = ½(0.04) = 0.02
  2. ∂L/∂ŷ = −(y − ŷ) = −(0.2) = −0.2
  3. Sigmoid deriv: σ'= ŷ(1 − ŷ) = 0.8 × 0.2 = 0.16
  4. ∂L/∂z = ∂L/∂ŷ × σ'= −0.2 × 0.16 = −0.032
  5. This gradient propagates backward, multiplied by each weight to update upstream layers (chain rule).
Plain English
Backprop is like tracing blame backward through a network. When the output is wrong, you figure out how much each neuron contributed to the error, and nudge its weights to be less wrong next time — starting from the output and working back to the input.
Interactive: Activation Function Plots
When to Use Each Activation
Sigmoid: Output layer for binary classification (gives 0–1 probability).
ReLU: Default for hidden layers — fast, avoids vanishing gradient.
Tanh: Hidden layers when you need −1 to 1 output (RNNs).
Softmax: Output layer for multi-class (sums to 1).
❌ Sigmoid/Tanh in deep hidden layers → vanishing gradient problem.
3
What is the vanishing gradient problem? Why does it occur with sigmoid activations in deep networks? Name two solutions.
⌄
Answer
Sigmoid derivative: σ'(z) = σ(z)(1−σ(z)) ∈ (0, 0.25]. In a 10-layer network, gradients are multiplied 10 times — 0.25¹⁰ ≈ 0.000001. Early layers receive nearly zero gradient → they barely update → model fails to learn.
Plain English
Like a game of telephone — each person (layer) whispers a quieter message. By the time the correction signal reaches the first layers, it's basically silence. The front layers don't learn.
Two Solutions
1. ReLU activation — derivative is 1 for z>0, so no shrinking across layers.
2. Residual connections (ResNet) — skip connections provide a gradient highway that bypasses layers.

🔄 RNN / LSTM Sequences · Vanishing Gradient · Gates
6
Draw and explain an unrolled RNN over 3 time steps. Why do RNNs suffer from vanishing gradient? How does LSTM solve this?
⌄
Unrolled RNN — 3 Time Steps
RNN Hidden State Equation
Symbol Breakdown
SymbolMeaning
hₜHidden state at time t — the "memory" passed forward
xₜInput at time step t (e.g., word embedding)
WₕWeight matrix for hidden-to-hidden connection
WₓWeight matrix for input-to-hidden connection
tanhActivation keeping values in [−1, 1]
Vanishing Gradient in RNNs
Gradient at step t must be multiplied by Wₕ and tanh' at every step back. Over 100 steps, if |Wₕ·tanh'| < 1, gradient → 0. The early steps learn nothing from long-range context.
LSTM Solution: Gates
LSTM adds a cell state (Cₜ) — a conveyor belt that carries information unchanged, modified only by multiplicative gates:

Forget gate (f): σ(Wf·[hₜ₋₁, xₜ]) — what to erase from cell state.
Input gate (i): σ(Wi·[hₜ₋₁, xₜ]) — what new info to write.
Output gate (o): σ(Wo·[hₜ₋₁, xₜ]) — what to expose as hidden state.

Gradients flow through cell state with minimal multiplication → no vanishing.
Plain English
RNN has a leaky short-term memory — old memories fade. LSTM is like writing on a whiteboard with a dedicated eraser (forget gate) and a separate marker (input gate). Important facts written on the board persist, while irrelevant ones get erased.
🎯 Attention & Transformers QKV · Self-Attention · Multi-Head
7
Write the scaled dot-product attention formula. What do Q, K, V represent? Given Q·Kᵀ produces scores [[4, 2],[1, 3]] with dₖ=2, compute the softmax attention weights.
⌄
The Formula
Symbol Breakdown
SymbolMeaningIntuition
QQuery matrix"What am I looking for?" — current token's question
KKey matrix"What do I contain?" — each token's advertisement
VValue matrix"What do I share?" — actual content to pass forward
dₖDimension of key vectorsScaling prevents large dot products → saturated softmax
√dₖScaling factorWithout this, softmax gradients become tiny for large dₖ
softmaxConverts scores to probability weights (sum=1)—
Calculation
  1. Scale: scores / √2 = [[4/1.41, 2/1.41],[1/1.41, 3/1.41]] ≈ [[2.83, 1.41],[0.71, 2.12]]
  2. Row 1 softmax: e^2.83/(e^2.83+e^1.41) = 16.9/(16.9+4.1) ≈ [0.80, 0.20]
  3. Row 2 softmax: e^0.71/(e^0.71+e^2.12) = 2.03/(2.03+8.33) ≈ [0.20, 0.80]
Plain English
Attention is like a search engine inside the model. Each word submits a query ("I need context about motion"). Other words broadcast their keys ("I describe motion"). High dot-product = strong match = high attention weight. Then you retrieve the values from matched words and mix them in proportion to the weights.
Self-Attention vs Cross-Attention
Self-attention: Q, K, V all come from same sequence. Used in encoder — each word attends to every other word in the same sentence.
Cross-attention: Q from decoder, K and V from encoder. Lets decoder focus on relevant input words when generating each output token.

Describe the GAN training process. Write the minimax loss. What are mode collapse and training instability?
⌄
GAN Minimax Loss
Symbol Breakdown
SymbolMeaning
GGenerator — learns to create fake samples from noise z
DDiscriminator — learns to distinguish real from fake

GAN Architecture Diagram
Mode Collapse
Generator finds a single output that consistently fools D, and produces only that — losing diversity. Example: trained on faces but only generates one face type.
Training Instability
If D gets too good, G's gradient → 0 (can't learn). If G gets too good, D gives useless gradients. Balance is delicate. Solutions: Wasserstein GAN (WGAN), gradient penalty, spectral normalization.
Plain English
GAN is a counterfeiter (G) vs. detective (D). The counterfeiter gets better by fooling the detective. The detective gets better by studying both real and fake bills. They improve each other — until the fakes are indistinguishable from real.
10
How does a VAE differ from a standard autoencoder? What is the reparameterization trick and why is it needed?
⌄
VAE Loss
Symbol Breakdown
SymbolMeaning
L_reconReconstruction loss — how well decoder recreates input
KLKL divergence — forces latent space toward Gaussian prior
μ, σMean and std predicted by encoder for the latent distribution

Reparameterization Trick
Sampling is not differentiable — backprop can't pass through a random node. Solution: sample ε from N(0,1) (fixed randomness), then compute z = μ + σε. Now gradients flow through μ and σ cleanly.
Plain English
Standard AE encodes to a single point — no structure in latent space. VAE encodes to a region (distribution) — nearby points decode to similar outputs. The trick moves the randomness outside the computation graph so the network can still learn.
🎮 Reinforcement Learning Q-Learning · Bellman · Policy Gradient
11
Write the Q-learning update rule (Bellman equation). An agent is in state s, takes action a, gets reward R=5, transitions to s'. Q(s,a)=3, max Q(s',a')=10, α=0.1, γ=0.9. Compute the new Q(s,a).
⌄
Bellman Update Equation
Symbol Breakdown
SymbolMeaningValue
Q(s,a)Current estimate of value of taking action a in state s3
αLearning rate — how fast to update Q values0.1
RImmediate reward received after taking action5
γDiscount factor — how much future rewards are valued0.9
max Q(s',a')Best Q-value possible from next state (greedy)10
TD targetR + γ·maxQ(s',a') — what we think Q(s,a) should be5+9=14
TD errortarget − current = how wrong we were14−3=11
Calculation
  1. TD target = R + γ · max Q(s',a') = 5 + 0.9 × 10 = 14
  2. TD error = 14 − 3 = 11
  3. New Q(s,a) = 3 + 0.1 × 11 = 4.1
Interactive Q-Learning Calculator
New Q(s,a) = 4.1
Plain English
Q(s,a) is a cheat sheet: "if I'm in situation s and do action a, I expect this much total future reward." The Bellman update says: "look what actually happened (R) plus the best I can hope for from here (γ·maxQ) — nudge my estimate toward that."
γ Intuition
γ=0: only care about immediate reward (shortsighted).
γ=1: weight future rewards equally to now (patient, but can diverge).
γ=0.9: typical — future reward discounted 10% per step.

15
A dataset has two concentric circles (inner class 0, outer class 1). Why does K-Means fail? Name two algorithms that handle this and explain why they work.
⌄
Concentric Circles Dataset
Why K-Means Fails
K-Means partitions by Euclidean distance to centroids — it creates linear/convex boundaries. Both centroids end up in the middle, splitting by left/right or up/down, not inner/outer.
Solution 1: DBSCAN
Clusters by density — groups points with enough neighbors within radius ε. Inner circle = one dense region; outer ring = another. Works because it doesn't assume convex shapes.
Solution 2: Kernel SVM / Spectral Clustering
Map to higher-dimensional space where circles become linearly separable (RBF kernel), then cluster there. Spectral clustering uses graph Laplacian eigenvectors to find non-convex clusters.

Compare L1 (Lasso) and L2 (Ridge) regularization. Write their loss functions. When does L1 produce sparse weights?
⌄
Regularized Loss Functions
Symbol Breakdown
SymbolMeaning
LOriginal loss (MSE, cross-entropy, etc.)
λRegularization strength — higher = more penalty on large weights
Σ|wⱼ|L1 penalty — sum of absolute values. Geometry: diamond shape → corners at axes → sparsity
Σwⱼ²L2 penalty — sum of squares. Geometry: sphere → touches axes at smooth curve → no sparsity
Why L1 Creates Sparsity
The L1 constraint region (diamond) has corners exactly on the axes. The loss function's minimum is most likely to touch a corner where one or more weights = 0 exactly. L2 (sphere) is smooth — minimum touches at a point with small but non-zero weights.
Plain English
L2 shrinks all weights toward zero proportionally (everyone gets a pay cut). L1 can cut some weights to exactly zero (fire some workers entirely). Use L1 when you suspect many features are irrelevant — it automatically does feature selection.
19
What is PCA? Given a 2D dataset, describe the steps to compute the first principal component. What does the explained variance ratio mean?
⌄
PCA Objective
Steps
  1. Center the data: subtract the mean from each feature
  2. Compute the covariance matrix: C = (1/n)XᵀX
  3. Compute eigenvalues (λ) and eigenvectors (v) of C
  4. Sort by eigenvalue descending — largest λ = most variance explained
  5. Project data onto top-k eigenvectors: Z = XV_k
Explained Variance Ratio
EVR for component i = λᵢ / Σλⱼ. If PC1 has EVR=0.85, it captures 85% of total variance. Choose k components to retain ≥95% of variance.
Plain English
PCA finds the directions (axes) along which the data spreads out most. The first axis captures the most variation. Think of projecting a 3D point cloud onto a 2D plane oriented for maximum spread — you lose minimal information.
When NOT to Use PCA
❌ When features are not correlated (PCA won't help much)
❌ When interpretability matters (principal components mix features)
❌ Before tree-based models (they handle high dimensions well anyway)

What is cross-entropy loss? When is it preferred over MSE for classification? Compute cross-entropy for ŷ=[0.7, 0.2, 0.1], true class=0.
⌄
Cross-Entropy Loss
Symbol Breakdown
SymbolMeaning
yᵢTrue label (one-hot: 1 for correct class, 0 otherwise)
ŷᵢPredicted probability for class i
logNatural log — penalizes very wrong confident predictions severely
−ΣNegative sum (log of probability < 1 is negative, so we negate)
Calculation
True class = 0, so y = [1, 0, 0]
Loss = −(1·log(0.7) + 0·log(0.2) + 0·log(0.1)) = −log(0.7) ≈ 0.357
Why Not MSE for Classification?
MSE treats all errors equally. A prediction of 0.49 vs 0.51 (wrong class) gets same weight as 0.01 vs 0.99. Cross-entropy penalizes confident wrong predictions exponentially — much stronger gradient signal for learning.

22
Explain dropout regularization. If a neuron has output 0.8 with dropout rate p=0.5 during training, what is the expected output at test time?
⌄
Answer
During training: each neuron independently zeroed out with probability p=0.5. Remaining neurons scaled by 1/(1−p) = 2 (inverted dropout) to keep expected value the same.

Training output (if not dropped): 0.8 × 2 = 1.6
Expected: 0.5 × 1.6 + 0.5 × 0 = 0.8

At test time: use all neurons, no scaling needed (already handled by training scaling).
Plain English
Dropout is like studying with random teammates missing — you can't rely on any one teammate, so you learn to do the job yourself. Each neuron learns robust features independently. At test time, the whole team shows up but each member contributes at half intensity (or equivalently, training scaled up so test needs no change).
