Softmax — choosing the predicted class. A network outputs softmax probabilities: Cat = 0.20, Dog = 0.65, Bird = 0.15. What class will the network predict, and why?
⌄

  1. Look at the probability for each class.
  2. Pick the class with the highest probability.
  3. The highest is 0.65 → Dog.
Predicted class: Dog
💡 Why
A softmax output layer gives class probabilities that sum to 1. The model predicts the class with the highest probability — no other arithmetic needed.

Softmax — what it does. Raw scores: Red = 4, Blue = 2, Green = 1. Without computing exact softmax probabilities, which class will get the highest probability? Why?
⌄

Red has the highest softmax probability

S3
Cross-Entropy
Cross-entropy — good vs bad prediction. True class is Positive. Model A predicts P(Positive)=0.90, P(Negative)=0.10. Model B predicts P(Positive)=0.40, P(Negative)=0.60. Which has lower cross-entropy loss? (No logarithms needed.)
⌄
📐 Steps
  1. Cross-entropy looks only at the probability assigned to the correct class.
  2. The correct class is Positive.
  3. Model A → 0.90 on Positive · Model B → 0.40 on Positive.
  4. Higher probability on the true class = lower loss.
Model A has lower cross-entropy loss
💡 Why
Cross-entropy = −log(p_true_class). The more confidently right you are, the smaller −log(p) becomes (and log(1) = 0). A confident-but-correct model is barely penalized; a wishy-washy or wrong model is heavily penalized.
Slide: "If the predicted probability for the true class is high, close to 1, error will be very low (because ln(1)=0)."

S5
ReLU
ReLU — which neurons activate? A hidden layer with ReLU has four neurons with net inputs −3, 0, 2, 5. Which neurons are active?
⌄
📐 Steps
  1. ReLU outputs 0 for inputs ≤ 0.
  2. ReLU keeps positive inputs unchanged.
  3. Apply: −3 → 0 (inactive), 0 → 0 (inactive), 2 → 2 (active), 5 → 5 (active).
Active neurons: inputs 2 and 5
💡 Why
ReLU is a piecewise linear gate — only positive inputs pass through. This produces sparse activations (typically only a fraction of neurons fire), which is computationally efficient and helps avoid the vanishing-gradient problem of sigmoid/tanh.
Slide: "Note that ReLU is 0 for half of its domain values."

S6
Dying ReLU
Dying ReLU. During training, one ReLU neuron has negative input for every training example and outputs 0 every time. Why is this a problem?
⌄
📐 Steps
  1. ReLU outputs 0 for all negative inputs.
  2. If a neuron is always at output 0, its gradient is also 0 (the ReLU slope is 0 there).
  3. Zero gradient means the neuron's weights don't get updated.
  4. The neuron may stay dead forever — adding no value to the network.
A dead ReLU neuron stops learning permanently and wastes capacity
💡 Why
A dead neuron is a stuck hidden unit. The network effectively becomes smaller. Causes include bad weight initialization or too-large learning rates that push pre-activations into the negative region.
Slide: "If a neuron does not activate for any training example in an iteration then it will never become active again."

S7
Leaky ReLU
Leaky ReLU. Many ReLU neurons in a network have stopped activating. Which activation helps reduce this — standard ReLU or Leaky ReLU? Why?
⌄
📐 Steps
  1. Standard ReLU outputs exactly 0 for negative inputs (and 0 gradient).
  2. Leaky ReLU outputs a small non-zero value (e.g., 0.01·x) for negative inputs.
  3. Because the gradient is non-zero in the negative region, weights can still update.
  4. Dead neurons can recover.
Use Leaky ReLU
💡 Why
Leaky ReLU breaks the "trap" of the dying ReLU by allowing a small signal through even when inputs are negative — so the neuron always has a path to recover via gradient updates.
Slide: "Prevents dying ReLUs problem."

S8
CNN vs Dense
CNN vs dense layer. You're classifying large images. Important patterns (edges, objects) can appear anywhere. Why is a CNN better than a fully connected dense network?
⌄
📐 Steps
  1. Images have spatial structure — nearby pixels matter together.
  2. CNN filters scan the image locally with shared weights.
  3. The same filter detects the same pattern anywhere in the image (translation equivariance).
  4. Dense layers treat every pixel as a unique input — they don't know that a cat in the corner is the same kind of object as a cat in the center.
CNN wins via local connections + shared weights + spatial structure
💡 Why
A 1000×1000 image fed to a dense layer has ~1 billion weights just in the first layer. A CNN with a 3×3 filter uses only 9 weights for that filter, reused across the whole image — dramatically more efficient and inductively biased toward the right structure.
Slide: "CNN is a type of deep learning architecture designed to work with grid-like input."

Max Pooling
Max pooling. A 2 × 2 region has values [[1, 4], [2, 3]]. What value is passed forward by max pooling?
⌄
📐 Steps
  1. Max pooling outputs the maximum value of the region.
  2. Values in the region: 1, 4, 2, 3.
  3. Maximum is 4.
Pass forward: 4
💡 Why
Max pooling keeps the strongest activation from a region. This gives translation invariance (exact position within the window stops mattering) and shrinks the feature map for computational savings. Average pooling is an alternative that smooths instead.

Transfer Learning
Transfer learning scenario. You want to classify horses, zebras, and mules. You have only a few hundred labeled images. Train a large CNN from scratch or use a pretrained CNN and fine-tune? Why?
⌄
📐 Steps
  1. A large CNN from scratch needs huge labeled data (~millions of images).
  2. A pretrained CNN already knows generic image features — edges, textures, shapes.
  3. Fine-tuning just adapts the top layers to your small new dataset.
  4. With only a few hundred images, scratch training will overfit immediately; fine-tuning will work.
Use a pretrained CNN and fine-tune
💡 Why
Transfer learning reuses already-learned generic features. The bottom CNN layers (edge detectors, color blobs, texture patches) are reusable across nearly any image task. Only the high-level classification head needs to learn the new task.

Early Stopping
Early stopping scenario. Training error keeps decreasing, but validation error starts increasing. What's happening, and what should you do?
⌄
📐 Steps
  1. Training error decreasing → the model is fitting the training data better.
  2. Validation error increasing → it's getting worse on unseen data.
  3. This is overfitting.
  4. Stop training and keep the model from the epoch with the lowest validation error.
Overfitting → use early stopping; save the best-validation model
💡 Why
Beyond a certain point, the network starts memorizing noise specific to the training set. Early stopping freezes the model at its best generalization snapshot — a free regularization technique with no extra hyperparameters except patience.
Slide: "Stop training when additional epochs actually increase validation error."

Dropout
Dropout scenario. A network does very well on training data but poorly on validation. How can dropout help?
⌄
📐 Steps
  1. Big train-vs-validation gap = overfitting.
  2. Dropout randomly removes a fraction of neurons during each training pass.
  3. This stops the network from depending too heavily on any one neuron.
  4. The network learns more redundant, robust patterns and generalizes better.
Dropout reduces overfitting by forcing the network to learn robust patterns
💡 Why
Dropout acts like training an implicit ensemble of thinned sub-networks — each training pass uses a different random subset. At test time, all neurons are present (with rescaling), giving an average over many sub-models.
Slide: "Dropout: Each time we load a training example, drop a random set of neurons from hidden and input layers."

Attention
Attention — context dependence. "The animal did not cross the street because it was tired." Why is attention useful when computing the representation of the word "it"?
⌄
📐 Steps
  1. "It" by itself has no meaning — it depends on what it refers to.
  2. The model needs to connect "it" to "animal" earlier in the sentence.
  3. Attention lets each word's representation pull information from every other word.
  4. The representation of "it" can be shaped strongly by "animal."
Attention links "it" to "animal" — context resolves pronoun reference
💡 Why
Static word embeddings give every "it" the same vector. Self-attention produces a context-dependent representation, so the same word in different sentences gets different vectors based on what it relates to.
Slide: "New representation of each word is influenced by all other words."

S15
Encoder-Decoder Attention
Self vs encoder-decoder attention. In a translation Transformer (English → Spanish), where do queries come from during encoder-decoder attention, and where do keys and values come from?
⌄
📐 Steps
  1. Encoder-decoder attention is the bridge between the encoder and decoder.
  2. The decoder is producing the next output word and needs to ask: "which input words matter for this output?"
  3. Queries (the question) come from the decoder.
  4. Keys and values (the answer pool) come from the encoder output.
Q ← decoder · K, V ← encoder
💡 Why
This is how the decoder selectively reads the encoded input. Compare with self-attention, where Q, K, V all come from the SAME sequence — that's used within the encoder (every input word attending to every other input word) and within the decoder (each output word attending to earlier output words).
Slide: "Encoder-decoder attention: The queries come from the previous decoder layer and keys and values come from the last encoder layer."

RL scenario. A robot learns to walk. It gets no correct-action label per step — only rewards depending on whether it balances or falls. Why is this reinforcement learning, not supervised learning?
⌄
📐 Steps

Reinforcement learning — agent learns from rewards, not labeled actions

Reward is a much weaker signal than an explicit label, but it applies to a much wider class of problems — anywhere we can measure the result of a sequence of actions but don't know the optimal action in advance (games, robotics, dialogue).

S17
Q-Values
Choosing the best action. In state s the Q-values are: Q(s, Left) = 5, Q(s, Right) = 2, Q(s, Jump) = 4. Which action does an optimal policy choose?
⌄
📐 Steps
  1. An optimal policy picks the action with the highest Q-value.
  2. Compare: 5 (Left), 2 (Right), 4 (Jump).
  3. Highest is 5 → Left.
Choose Left
💡 Why
Q(s, a) is the estimated total future reward from doing action a in state s and then acting optimally. The optimal policy is simply π*(s) = argmax_a Q(s, a). During training the agent might explore (ε-greedy) — but the greedy choice always picks the highest Q.
Slide: "Take that action, a, from state s which has the highest Q(s, a)."

GAN Training
GAN training dynamics. The generator currently produces fakes that the discriminator catches easily. What should happen as training continues?
⌄
📐 Steps
  1. The discriminator improves at telling real from fake.
  2. The generator improves at producing more realistic fakes.
  3. The two networks compete — their goals are opposite.
  4. Over time, the generator's fakes become harder to detect.
Generator improves to fool the discriminator; discriminator keeps improving in response
💡 Why
GAN training is adversarial: a minimax game. Equilibrium is reached when the generator produces samples indistinguishable from real. Practical risks include mode collapse (generator settles on one good fake) and instability if the discriminator is too strong too early.
Slide: "Generator gradually learns to generate more and more realistic images to dupe the discriminator!"

Autoencoder
Autoencoder bottleneck. An autoencoder is trained with input = target. The middle (bottleneck) layer is much smaller than the input. What is the purpose of this smaller middle layer?
⌄
📐 Steps
  1. The autoencoder must reconstruct the input from the bottleneck.
  2. Because the bottleneck is smaller than the input, it can't copy everything directly.
  3. It must learn to compress — keep what's essential, drop what's redundant.
  4. The result is a compact representation of the input.
The bottleneck forces a compact, learned representation
💡 Why
Without the bottleneck, an autoencoder could just learn the identity function and be useless. The narrow waist is the regularizer — it forces the encoder to discover an efficient code. That code (the bottleneck activations) is a learned feature vector useful for downstream tasks, anomaly detection, denoising, etc.
Slide: "The bottleneck layer needs to capture all the useful information in the input to be able to reconstruct it."

Q1
Perceptron
Perceptron update. Given w = [0.5, −0.3], bias b = 0.1, input x = [1, 2], true label y = 1, learning rate η = 0.2. Compute the predicted output, decide whether an update is needed, and find the new weights and bias.
⌄
Given w = [0.5, −0.3] · b = 0.1 · x = [1, 2] · y = 1 · η = 0.2 · Step activation: ŷ = 1 if z > 0, else 0.
🖼️ Visual
📐 Steps (with equations)
  1. Net input:
    z = (0.5)(1) + (−0.3)(2) + 0.1 = 0.5 − 0.6 + 0.1 = 0
  2. Step activation: z = 0 is not strictly > 0, so ŷ = 0.
  3. Error: y − ŷ = 1 − 0 = 1. Non-zero → update needed.
  4. Weight update rule:
    w₁ ← 0.5 + 0.2 · 1 · 1 = 0.7 w₂ ← −0.3 + 0.2 · 1 · 2 = 0.1 b ← 0.1 + 0.2 · 1 = 0.3
Final: w = [0.7, 0.1], b = 0.3
💡 Tip — Solve Without Equations
  1. Multiply each weight by its input, then add everything together (including bias). That's z.
  2. Did z come out positive or zero/negative? Positive → ŷ=1. Otherwise → ŷ=0.
  3. Did the answer match y? If yes, stop. If no, the perceptron was "off by 1" in one direction.
  4. To fix it, nudge each weight by (η · error · input). Treat η as the size of the nudge, and the input as how much that weight contributed.
  5. The bias gets the same nudge but without the input multiplier (you can imagine the bias has its own input that is always 1).
Mental check: error = +1 means we predicted too low → push weights UP for positive inputs (which we did).
🧠 Why It Works
The error (y − ŷ) drives the update direction. Inputs that were "on" (positive xⱼ) get their weights pushed in the error's direction. The perceptron convergence theorem guarantees this rule finds a separating line, if one exists.

Q2
CNN
CNN output size + parameter count. Input image: 32 × 32 × 3 (RGB). A convolutional layer applies 16 filters of size 5 × 5, with stride 1 and zero padding. Then a 2 × 2 max-pool layer with stride 2 follows. (i) Find the output spatial size after each layer. (ii) Count the parameters in the conv layer (include bias).
⌄
Formulas (memorize, not given) Output = ⌊(W − F + 2P) / S⌋ + 1 · Params = (F · F · Cᵢₙ + 1) × Cₒᵤₜ
🖼️ Visual — spatial shrinking
📐 Steps (with equations)
  1. Output size formula:
  2. Conv: W=32, F=5, P=0, S=1. ⌊(32 − 5 + 0) / 1⌋ + 1 = 27 + 1 = 28 → 28 × 28 × 16 feature maps
  3. Max-pool: W=28, F=2, P=0, S=2. ⌊(28 − 2) / 2⌋ + 1 = 13 + 1 = 14 → 14 × 14 × 16
  4. Parameter formula:
  5. Plug in F=5, Cᵢₙ=3, Cₒᵤₜ=16: (5·5·3 + 1) × 16 = (75 + 1) × 16 = 76 × 16 = 1,216
Conv output: 28×28×16 · Pool output: 14×14×16 · Conv params: 1,216
💡 Tip — Solve Without Equations
  1. For "valid padding" (P=0, S=1): output = input − filter + 1. Just subtract filter size and add 1. (32 − 5 + 1 = 28.)
  2. For 2×2 max-pool stride 2: output = input / 2. (28 / 2 = 14.) Pooling halves the spatial size.
  3. For parameter count: imagine one filter — it has filter_size × filter_size × input_channels weights plus 1 bias. Multiply by the number of filters.
    • One filter = 5 × 5 × 3 + 1 = 76 weights.
    • 16 filters total = 76 × 16 ≈ 1,200 (close enough mental math).
  4. For "same padding": output size stays equal to input. Use P = (F−1)/2.
Common mistake: forgetting the input channels in the filter (each filter has depth = input channels). Also forgetting the bias.
🧠 Why It Works
Each filter spans all input channels — that's why depth = 3 enters the parameter count. Weight sharing across positions is what keeps CNNs small: a dense layer on a 32×32×3 input would need ~3,000 weights per output unit; this conv layer needs just 76 weights per filter regardless of input spatial size.

Q3
CNN
Multi-layer CNN spatial size. Input 64 × 64. Apply Conv(F=3, S=1, P=1) → MaxPool(2×2, S=2) → Conv(F=3, S=1, P=0) → MaxPool(2×2, S=2). What is the final feature-map spatial size?
⌄
📐 Steps
  1. Conv 1 (F=3, S=1, P=1, same padding): ⌊(64 − 3 + 2)/1⌋ + 1 = 62 + 1 = 64 (size preserved).
  2. Pool 1: ⌊(64 − 2)/2⌋ + 1 = 31 + 1 = 32 (halved).
  3. Conv 2 (F=3, S=1, P=0, valid): ⌊(32 − 3)/1⌋ + 1 = 29 + 1 = 30 (shrinks by 2).
  4. Pool 2: ⌊(30 − 2)/2⌋ + 1 = 14 + 1 = 15.
Final spatial size: 15 × 15
💡 Tip — Solve Without Equations
  1. Same padding + stride 1: size doesn't change. (Easy — just copy the input size.)
  2. Pool 2×2 stride 2: halve the size. (64 → 32, 32 → 16, etc.)
  3. Valid padding F=3 stride 1: shrinks by 2 (F − 1). So 32 → 30.
  4. Stack them mentally: 64 → (same) → 64 → (halve) → 32 → (−2) → 30 → (halve) → 15.
Trick: "same" preserves, "pool 2×2 s=2" halves, "valid F=3" subtracts 2. Most CNN size questions are just these three operations chained.
🧠 Why It Works
"Same padding" pads the input so the filter can be centered on every position, including edges. Without padding, the filter loses (F−1) pixels along each dimension. Pooling halves the size because stride equals the window width.

Q4
Backprop
Backprop with sigmoid output. A network with sigmoid output predicts ŷ = 0.8 for a training example with true label y = 1. Using MSE loss L = ½(y − ŷ)², compute the loss, the derivative dL/dŷ, the sigmoid derivative σ'(z), and the gradient dL/dz flowing back from the output.
⌄
📐 Steps
  1. Loss: L = ½(1 − 0.8)² = ½(0.04) = 0.02.
  2. dL/dŷ = −(y − ŷ) = −(0.2) = −0.2. (The negative sign matters — predicted is below target, so increasing ŷ decreases loss.)
  3. Sigmoid derivative at the output: σ'(z) = ŷ · (1 − ŷ) = 0.8 · 0.2 = 0.16.
  4. Chain rule: dL/dz = (dL/dŷ) · σ'(z) = −0.2 · 0.16 = −0.032.
L = 0.02 · dL/dŷ = −0.2 · σ'(z) = 0.16 · dL/dz = −0.032
💡 Tip — Solve Without Equations
  1. MSE loss intuition: just half the squared error. (1 − 0.8)² = 0.04, half = 0.02.
  2. Loss derivative dL/dŷ: equals −(target − prediction). Predicted too low → negative gradient → push prediction up.
  3. Sigmoid derivative trick: if you already know ŷ (sigmoid output), then σ'(z) = ŷ · (1 − ŷ). No need to compute z. Here ŷ = 0.8 → σ' = 0.8 × 0.2 = 0.16.
  4. Chain rule: multiply the two pieces together.
Sanity check: gradient is negative because we want to increase the prediction (it was too low).
🧠 Why It Works
The sigmoid derivative is tiny (max 0.25 at z = 0). Multiplying many such terms across layers shrinks the gradient toward zero — that's the vanishing gradient problem. ReLU's derivative is 1, which is why deep nets use it instead.

Q5
Softmax + CE
Softmax + cross-entropy (values given). A 3-class classifier outputs logits z = [2, 1, 0]. The true class is class 0. Given values: e² ≈ 7.4, e¹ ≈ 2.7, e⁰ = 1. (i) Find the softmax probabilities. (ii) Find the cross-entropy loss using the given value ln(0.66) ≈ −0.42.
⌄
Given values (no calculator needed) e² ≈ 7.4 · e¹ ≈ 2.7 · e⁰ = 1 · ln(0.66) ≈ −0.42
🖼️ Visual — what softmax does
Softmax exponentiates, then normalizes — biggest logit wins, but smaller logits still get some probability.
📐 Steps (with equations)
  1. Softmax formula:
  2. Plug the given exponentials in: numerators: e² = 7.4, e¹ = 2.7, e⁰ = 1 sum: 7.4 + 2.7 + 1 = 11.1
  3. Divide each numerator by the sum: p₀ = 7.4 / 11.1 ≈ 0.66 p₁ = 2.7 / 11.1 ≈ 0.24 p₂ = 1 / 11.1 ≈ 0.09 Sanity check: 0.66 + 0.24 + 0.09 ≈ 1.0 ✓
  4. Cross-entropy formula (one-hot label y = [1, 0, 0]):
    L = −ln(p₀) = −ln(0.66) = −(−0.42) = 0.42
p ≈ [0.66, 0.24, 0.09] · L ≈ 0.42
💡 Tip — Solve Without Equations
  1. Which class wins? Look only at the logits — biggest logit becomes biggest probability. Here z₀=2 is the largest, so class 0 wins. (You don't need to compute the actual numbers to answer "which class".)
  2. Rough probability split: if the top logit is much larger than the others, expect the top probability to be close to 1. If logits are similar, expect roughly equal probabilities.
  3. Cross-entropy loss intuition: CE only cares about the probability assigned to the TRUE class.
    • If true-class probability is close to 1 → loss is close to 0 (good).
    • If true-class probability is close to 0 → loss is huge (bad).
  4. Here: p₀ ≈ 0.66 for the true class → small-ish loss (0.42). If p₀ were 0.99, loss would be near 0.
Quick check on an exam: exam will ALWAYS give you the e^x or ln values needed — no calculator. Just plug-and-chug.
🧠 Why It Works
Exponentiating exaggerates gaps between scores (e² is much bigger than e⁰, even though 2 and 0 differ by only 2). Normalizing by the sum makes all outputs add to 1, so they look like probabilities. Cross-entropy then measures how concentrated those probabilities are on the correct class.

Vanishing Gradient
Vanishing gradient (small numbers). Each sigmoid layer contributes σ'(z) = 0.25 (its maximum value) to the gradient. After 4 stacked sigmoid layers, what is the product of the four derivatives? Compare to a 4-layer ReLU network where every neuron is active (derivative = 1).

Given σ'(z) = 0.25 per sigmoid layer · ReLU'(z) = 1 when active · 4 stacked layers
🖼️ Visual — gradient shrinking through layers
Each sigmoid layer multiplies the gradient by 0.25 → it shrinks fast. ReLU keeps the gradient intact.
📐 Steps (with equations)
  1. Chain rule for the gradient through 4 layers:
  2. Sigmoid case: product = 0.25 × 0.25 × 0.25 × 0.25 = (1/4)⁴ = 1/256 ≈ 0.004

Q10
Q-Learning
Q-Learning Bellman update. An agent in state s takes action a, receives reward R = 5, and transitions to s'. Current Q(s, a) = 3. The max Q-value over next-state actions is max_{a'} Q(s', a') = 10. Learning rate α = 0.1, discount γ = 0.9. Compute the new Q(s, a). Also compute the TD target and TD error.
⌄
📐 Steps
  1. TD target = R + γ · max_{a'} Q(s', a'): TD target = 5 + 0.9 · 10 = 5 + 9 = 14
  2. TD error = TD target − current Q: TD error = 14 − 3 = 11
  3. New Q = Q + α · TD error: New Q(s, a) = 3 + 0.1 · 11 = 3 + 1.1 = 4.1
TD target = 14 · TD error = 11 · New Q(s, a) = 4.1
💡 Tip — Solve Without Equations
  1. Three numbers to compute, in order:
    • TD target = "what I now think the value should be" = reward + γ × best next Q. (5 + 0.9·10 = 14.)
    • TD error = target − current = how wrong I was. (14 − 3 = 11.)
    • New Q = current + α × error = take a small step toward the target. (3 + 0.1·11 = 4.1.)
  2. Two parameters not to confuse: α = how fast you update; γ = how much you care about the future. Both are between 0 and 1.
  3. Quick sanity check: the new Q should be BETWEEN the old Q and the target. (3 < 4.1 < 14 ✓)
One-liner: "Bellman update = take a small step from old estimate toward (reward + best future)."
🧠 Why It Works
The TD target is what we now estimate Q(s, a) should be — based on what just happened (R = 5) plus the best future we expect (γ · 10). Old estimate was 3, off by 11. Small step (α = 0.1) toward the new estimate avoids reacting too strongly to single experiences.

RL — Multi-Step
Q-Learning over two steps. Q-table starts at Q(s, a) = 0 for all (s, a). Agent does: (1) takes action a₁ from s₀, receives R = 2, lands in s₁ (terminal, max Q(s₁, ·) = 0). (2) Episode 2: takes action a₁ from s₀ again, R = 2, lands in s₁. α = 0.5, γ = 0.9. Compute Q(s₀, a₁) after each episode.
⌄
📐 Steps
  1. Episode 1: Q(s₀, a₁) starts at 0. TD target = 2 + 0.9·0 = 2. TD error = 2 − 0 = 2. New Q(s₀, a₁) = 0 + 0.5 · 2 = 1.0
  2. Episode 2: Q(s₀, a₁) is now 1.0. Same transition. TD target = 2 + 0 = 2. TD error = 2 − 1.0 = 1. New Q(s₀, a₁) = 1.0 + 0.5 · 1 = 1.5
  3. The Q-value approaches the true value (= 2 for this deterministic case) but doesn't reach it in one update — the learning rate α = 0.5 limits each step. With more episodes Q → 2.
After Ep 1: Q = 1.0 · After Ep 2: Q = 1.5 · Limit: → 2.0
💡 Tip — Solve Without Equations
  1. Terminal state shortcut: if next state is terminal, the future-reward term is 0. TD target = just the reward.
  2. With α = 0.5: each update closes HALF the gap between current Q and target.
    • Q = 0 → target = 2 → halfway is 1.
    • Q = 1 → target = 2 → halfway is 1.5.
    • Q = 1.5 → target = 2 → halfway is 1.75.

🧠 Why It Works
Each update closes a fixed fraction (α) of the gap. The Q-value converges geometrically to the true return. Higher α = faster convergence but more sensitive to noise; lower α = slower but more stable.
