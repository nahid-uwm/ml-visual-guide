Neural Networks & Backprop
Perceptron → MLP → Deep Networks. Learn weights by gradient descent.
TL;DR: A neural net is just a chain of (multiply by weights → add bias → apply non-linearity). Training nudges weights downhill on a loss surface using gradients computed by the chain rule (backpropagation).
💡 The Big Idea
Each neuron computes z = w·x + b, then applies an activation function like ReLU or sigmoid. Stack neurons in layers — each layer's output is the next layer's input. The universal approximation theorem says one hidden layer with enough neurons can approximate any continuous function. We train by defining a loss (how wrong we are), computing ∂L/∂w for every weight via the chain rule, and stepping each weight in the direction that reduces loss.
🔑 Key Terms (Memorize)
Perceptron
Single neuron with step activation. Solves only linearly separable problems.
Activation Function
Non-linear function applied to z. Without it, layers collapse into one linear map.
Sigmoid σ(z)
1/(1+e⁻ᶻ). Range (0,1). Derivative ≤ 0.25 → vanishing gradient.
ReLU
max(0, z). Default for hidden layers. Derivative = 1 for z>0.
Softmax
Converts logits to probabilities that sum to 1. Used in multi-class output.
Backpropagation
Chain-rule application: gradients flow output → input through the computational graph.
Learning Rate η
Step size in weight update: w ← w − η·∂L/∂w. Too big = diverge, too small = slow.
Vanishing Gradient
In deep nets with sigmoid/tanh, gradients shrink to ~0, early layers don't learn.
Dropout
Randomly zero p% of neurons during training. Regularization, no extra params.
Batch Norm
Normalize activations per mini-batch. Enables higher LR, faster training.

  • Forgetting to add bias when computing z = w·x + b. Bias shifts the activation — without it, the line passes through origin.
    Always include +b in your manual calculation.
  • Confusing ŷ with z. z is the pre-activation; ŷ is after the activation function.
    For perceptron: ŷ = step(z). For sigmoid output: ŷ = σ(z).
  • Computing σ'(z) wrong. Many write σ'(z) = σ(z), missing the (1−σ(z)) factor.
    σ'(z) = σ(z)·(1−σ(z)). If you already computed σ(z)=0.8, then σ'=0.8×0.2=0.16.
  • Using MSE for classification. MSE gives weak gradients on confident-wrong predictions; cross-entropy gives strong correction signal.
    MSE for regression, cross-entropy for classification.
  • Applying step function with ≥ instead of >. The perceptron convention is ŷ=1 if z>0, ŷ=0 if z≤0. When z=0 exactly, output is 0.
    If z=0 and y=1, error = 1 → update IS needed.
  • Forgetting to update bias. The bias update is b ← b + η(y−ŷ), separate from weight updates.
    Same learning rule applies to b, just without the xᵢ multiplier.
⚡ Edge case XOR is the classic non-linearly-separable problem. A single perceptron cannot solve it — you need at least one hidden layer. This is the historical reason MLPs were invented.
🧠 Memory hook "Forward: z = Wx + b → activate → repeat. Backward: ∂L/∂w = upstream_gradient × local_gradient × input."

Convolutional Neural Networks
Share weights across image positions. Learn hierarchical features.
TL;DR: A CNN slides small learned filters across an image to detect patterns. Same filter weights apply at every position (weight sharing), so it learns once where an edge exists. Pooling then summarizes regions.
💡 The Big Idea
A dense layer on a 1000×1000 image would need ~10⁹ weights and treat every pixel as independent. A CNN's convolution layer uses a small (e.g. 3×3) filter, slid across the image, with the same 9 weights at every position. Early layers learn edges, middle layers learn textures, deep layers learn object parts. Pooling (max or average over a 2×2 region) shrinks spatial dimensions and adds small-shift invariance.
🔑 Key Terms
Filter / Kernel
Small matrix (F×F) of learned weights slid across input.
Stride S
How many pixels the filter moves each step. S=2 halves output size.
Padding P
Extra zero pixels at borders. "Same" padding preserves spatial size.
Feature Map
Output of a convolution. One per filter.
Receptive Field
Region of input that influences one output unit. Grows with depth.
Weight Sharing
Same filter weights reused at every spatial position.
Max Pooling
Take maximum in 2×2 window. No learnable parameters.
Translation Equivariance
Shifted input → shifted feature map. Conv has this; FC layers don't.
📐 The Two Formulas You Must Know
Output Spatial Size
Parameter Count Per Conv Layer
  • Forgetting the +1 in the output-size formula. A filter at position 0, 1, …, (W−F)/S occupies (W−F)/S + 1 positions.
    Output = ⌊(W − F + 2P) / S⌋ + 1. Always add 1.
  • Doubling padding by mistake. P=1 adds 1 pixel on each side → effective width is W + 2P, not W + P.
    "+ 2P" in the formula already accounts for both sides.
  • Forgetting the bias in parameter count. Each filter has F×F×Cᵢₙ weights PLUS 1 bias.
    Params = (F·F·Cᵢₙ + 1) × Cₒᵤₜ. For 3 in, 64 out, 3×3: (27+1)×64 = 1,792.
  • Confusing equivariance with invariance. Convolution is equivariant (shift in → shift out). Pooling adds the invariance.
    Equivariant = output shifts with input. Invariant = output unchanged.
  • Forgetting channels in 3D conv. For RGB input (3 channels), each 3×3 filter is actually 3×3×3 = 27 weights, not 9.
    Filter shape always includes input channels.
  • Treating pooling as having parameters. Max/average pooling has zero learnable weights.
    Only conv and FC layers contribute to param count.
⚡ Edge case — "Same" padding For F=3, stride=1: P=1 gives same output size as input. For F=5, stride=1: P=2. General rule: P = (F−1)/2 for odd F.
⚡ Edge case — Non-integer output If (W−F+2P)/S isn't an integer, you must floor it. The formula handles this with ⌊·⌋. Some frameworks instead pad to make it work — verify which convention your exam uses.
🧠 Memory hook "Stride shrinks. Padding grows. Filter eats. Bias adds one."

RNN / LSTM / GRU
Networks with memory. Process sequences one step at a time.
TL;DR: An RNN reuses the same weights at every time step and carries a hidden state forward as memory. LSTM adds gates and a cell-state highway to prevent gradients from vanishing across long sequences.
💡 The Big Idea
Plain RNN: hₜ = tanh(Wₕhₜ₋₁ + Wₓxₜ + b). Same weights at every step. Trained by backpropagation through time (BPTT) — unroll the network across time, compute gradients, sum updates. Problem: gradients multiplied by Wₕ at every step, so they vanish (|Wₕ|<1) or explode (|Wₕ|>1). LSTM solves this with a cell state Cₜ that flows through with minimal multiplications, plus three gates (forget, input, output) controlling what enters and exits.
🔑 Key Terms
Hidden State hₜ
The RNN's memory at time t, passed to next step.
Cell State Cₜ
LSTM-only. Long-term memory conveyor belt.
Forget Gate f
σ(·). Decides what to erase from cell state.
Input Gate i
σ(·). Decides what new info to add.
Output Gate o
σ(·). Decides what to expose as hidden state.
BPTT
Backprop Through Time. Unroll RNN, then standard backprop.
Gradient Clipping
Cap gradient norm to prevent explosion in RNNs.
Teacher Forcing
During training, feed true output as next input. Speeds convergence.

  • Forgetting RNN shares weights across time. It's NOT a different W per step.
    Same Wₕ, Wₓ at every time step. That's why param count is independent of sequence length.
  • Confusing vanishing with exploding gradient. Both happen in RNNs but require different fixes.
    Vanishing → LSTM/GRU or skip connections. Exploding → gradient clipping.
  • Claiming LSTM has no vanishing gradient. It reduces it dramatically, but doesn't eliminate it for very long sequences (1000+ steps).
    LSTM handles ~500 steps reliably; Transformers handle much longer.
  • Mixing up gate functions. Forget is for cell state, input is for new info, output is for hidden state.
    Mnemonic: Forget the past, Input the new, Output what to show.
  • Saying "RNN can't be parallelized at all." Within a batch, examples ARE parallel. The sequence dimension is the unparallelizable axis.
    Batch-parallel YES, time-parallel NO. This is why Transformers won.
⚡ Edge case — Bidirectional RNN Run one RNN left→right and one right→left, concatenate hidden states. Captures both past and future context but requires the full sequence upfront — useless for online generation.
🧠 Memory hook "Plain RNN forgets fast. LSTM's cell state is a conveyor belt. Three gates decide: erase, write, expose."

Attention & Transformers
Every token directly looks at every other token. Foundation of all modern LLMs.
TL;DR: Each token emits a Query, a Key, and a Value. The query is compared against all keys (dot product), softmaxed to get attention weights, then used to mix the values. This lets every token see every other in one step.
💡 The Big Idea
Imagine a library. Each book has a key (its topic) and a value (its content). When you (a token) want information, you submit a query. The library matches your query against every key (via dot product), scores the matches (softmax → probabilities), then gives you a weighted blend of values. Self-attention lets each token "ask the room a question" and get a context-aware answer in one step. Multi-head attention runs this in parallel with h different sets of (Q,K,V) projections to capture different types of relationships.
🔑 Key Terms
Query Q
"What am I looking for?" — projection of current token.
Key K
"What do I represent?" — advertisement of each token.
Value V
"What do I pass along?" — content shared if selected.
dₖ
Dimension of key vectors. We divide scores by √dₖ for stability.
Self-Attention
Q, K, V all from the same sequence (encoder).
Cross-Attention
Q from decoder, K/V from encoder. Decoder reads encoder.
Masked Self-Attention
Causal mask prevents looking at future tokens. Used in GPT.
Positional Encoding
Added to embeddings so model knows token order.
Multi-Head
h parallel heads with d/h dimensions each. Concatenate outputs.
📐 The Formula That Runs the World
Scaled Dot-Product Attention
  • Forgetting the √dₖ scaling. Without it, dot products grow large for big dₖ → softmax saturates → near-zero gradients.
    Always divide raw scores by √dₖ before softmax.
  • Applying softmax to the wrong axis. Softmax is over keys (each query distributes 1 unit of attention over all keys).
    For QKᵀ matrix of shape (n_query × n_key), softmax along the n_key axis (rows).
  • Saying "attention has no parameters." The Wq, Wk, Wv, Wo projection matrices ARE learned parameters.
    Q = X·Wq etc. The matrices are what's learned; the dot products are just operations.
  • Forgetting positional encoding. Pure attention is order-agnostic — "dog bites man" = "man bites dog" without PE.
    Add sinusoidal or learned positional encoding to embeddings BEFORE attention.
  • Confusing self-attention with cross-attention. Self: Q,K,V all from same sequence. Cross: Q from one, K,V from another.
    Encoder uses self. Decoder uses BOTH (masked self-attn over decoder, cross-attn to encoder).
  • Believing multi-head adds parameters. Each head has d/h dimensions. h heads × d/h = d, same total as single head.
    Param count of multi-head equals single-head — h is just how you split.
⚡ Edge case — O(n²) memory Attention is O(n²) in sequence length. A 4096-token context with d=512 needs ~67MB just for one attention matrix per head. Why context windows are expensive.
⚡ Edge case — Causal mask In GPT-style models, mask is an upper-triangular matrix of −∞. After softmax, those positions get weight 0. Token t cannot attend to t+1, t+2, …
🧠 Memory hook "Query asks. Keys advertise. Values deliver. Scale by √dₖ. Softmax over keys. Multiply by V."

Generative AI: GAN, VAE, Diffusion
Three ways to generate new data from learned distributions.
TL;DR: GAN = counterfeiter vs detective. VAE = autoencoder with smooth latent space. Diffusion = learn to denoise random noise back into images, step by step.
💡 The Big Idea
GAN: A Generator turns noise z into fake images. A Discriminator tries to tell real from fake. They train against each other — Generator improves to fool D, D improves to detect G. Equilibrium = G's fakes are indistinguishable.

VAE: An autoencoder where the encoder outputs distributions (μ, σ) instead of points. A KL-divergence term forces the latent space toward a Gaussian. Sampling z and decoding produces new examples. Smoother latents than AE, but blurrier outputs than GAN.

Diffusion: Take a clean image, add noise over 1000 steps until it's pure noise. Train a U-Net to predict and remove noise step by step. To generate: start from random noise, denoise 1000 times. No adversarial dynamics, very stable.
🔑 Key Terms
Generator G
Maps noise z to fake sample. Goal: fool D.
Discriminator D
Outputs P(real). Goal: D(real)→1, D(fake)→0.
Latent z
Random vector input to generator. Usually ~N(0, I).
Mode Collapse
G outputs only one type of sample. Loses diversity.
Minimax Loss
min_G max_D V(D,G). G minimizes, D maximizes simultaneously.
Reparameterization
VAE trick: z=μ+σ·ε, ε~N(0,I). Allows backprop through random sampling.
KL Divergence
Measure of distance between distributions. VAE regularizer.
Forward Process
Diffusion's noise-adding chain x₀ → xₜ.
Reverse Process
Diffusion's learned denoising xₜ → x₀.
⚖️ Compare: GAN vs VAE vs Diffusion

Has explicit likelihood❌ No✅ Approx (ELBO)✅ Yes

  • Confusing G's objective with D's. G wants D(G(z))→1 (fool D). D wants D(real)→1 AND D(G(z))→0.
    G minimizes log(1−D(G(z))); D maximizes log D(x) + log(1−D(G(z))).
  • Forgetting why we need reparameterization. You cannot backpropagate through "sample from N(μ,σ)" — it's a discrete stochastic op.
    Reparameterize as z = μ + σ·ε with ε~N(0,1). Now gradients flow through μ and σ.
  • Calling autoencoders "generative." A plain AE only reconstructs; its latent space has no smooth distribution to sample from.
    AE = compression. VAE = generation. The KL term is what makes VAE generative.
  • Thinking GANs are always better than VAEs. GANs produce sharper images but suffer mode collapse and unstable training; VAEs are reliable but blurry.
    Choose by use case: GAN for sharp images, VAE for stable training + interpretable latents.
  • Confusing diffusion's "noise prediction" with "image prediction." The network predicts ε (the noise added), not the clean image directly.
    Loss = ‖ε − ε_θ(xₜ, t)‖². Predicting noise is easier to train than predicting x₀.
  • Forgetting that diffusion sampling is slow. Generating one image from DDPM needs 1000 forward passes through the U-Net.
    DDIM, distillation, latent diffusion (Stable Diffusion) all aim to fix this.
⚡ Edge case — Conditional generation cGAN, cVAE, classifier-free guidance (in diffusion) all condition on a label y. Feed y as an extra input to G and D (cGAN), or to the U-Net (diffusion).
🧠 Memory hook "GAN = forger vs cop. VAE = compressor with a smooth dial. Diffusion = walk noise back to image."

Reinforcement Learning
Learn by trial and error. Maximize cumulative reward.
TL;DR: An agent observes state s, picks action a, gets reward r, lands in s′. Q(s,a) estimates the total future reward from doing a in s. The Bellman update gradually refines Q toward the true value.
💡 The Big Idea
RL frames problems as Markov Decision Processes (MDPs): states, actions, rewards, transitions. The agent's job is to find a policy π(a|s) that maximizes expected discounted return Σ γᵗrₜ. Q-learning estimates Q(s,a) — "if I do a in s, then act optimally afterwards, what total reward do I get?" The Bellman update bootstraps: new estimate = (1−α)·old + α·(reward + γ · best next Q). Repeat until convergence.
🔑 Key Terms
State s
Current situation observable to the agent.
Action a
Choice the agent makes from a set A(s).
Reward r
Scalar feedback from environment after action.
Policy π(a|s)
Strategy: maps states to action probabilities.
Q-value Q(s,a)
Expected return from (s,a) under optimal future policy.
Discount γ
∈ (0,1). How much to value future vs immediate reward.
Learning Rate α
Step size for Q updates. Different from γ.
TD Target
r + γ·maxQ(s′,a′). The improved estimate Q is pulled toward.
TD Error
TD target − current Q. The correction signal.
ε-greedy
Explore: random with prob ε. Exploit: argmax Q otherwise.

  • Confusing α (learning rate) with γ (discount). α is how fast Q updates. γ is how much you care about future.
    α = step size on Q. γ = weight on future. Both ∈ (0,1) but mean different things.
  • Computing TD target wrong. Many forget the max over a' or use the wrong Q.
    TD target = R + γ · max_{a'} Q(s', a'). Use NEXT state's BEST Q, not current state.
  • Forgetting to multiply by α. The update isn't Q ← TD target. It's Q ← Q + α·(TD target − Q).
    Equivalent: Q ← (1−α)·Q + α·(TD target). Both forms test the same thing.
  • Using Q-learning's max in SARSA. SARSA uses the actual next action's Q, not the max.
    Q-learning: max Q(s',·). SARSA: Q(s', a' actually taken by ε-greedy).
  • Saying Q-learning "doesn't explore." The behavior policy (ε-greedy) explores; the LEARNED policy is greedy.
    Q-learning learns the optimal Q regardless of exploration policy — that's what off-policy means.
  • Mixing up reward vs return. Reward = immediate r. Return = Σ γᵗrₜ cumulative.
    Q(s,a) estimates RETURN, not immediate reward.
⚡ Edge case — Setting γ = 0 γ=0 → agent only cares about immediate reward. γ=1 → all future rewards equally weighted; can diverge if episode is infinite.
⚡ Edge case — Terminal state When s' is terminal, there is no next state. The TD target collapses to just R (drop the γ·maxQ term).
🧠 Memory hook "Q-table is a cheat sheet: 'do a in s, expect this much future reward.' Bellman says: 'nudge old estimate toward what just happened + best you can hope for next.'"

Unsupervised Learning & Clustering
Find structure in data without labels. K-Means, HAC, DBSCAN, PCA.
TL;DR: K-Means = greedy iterative centroid update. HAC = bottom-up merging into a tree. DBSCAN = density-based, handles weird shapes. PCA = project onto directions of maximum variance.
💡 The Big Idea
K-Means: Pick K. Randomly place centroids. Assign each point to nearest centroid. Move each centroid to the mean of its cluster. Repeat. Minimizes within-cluster squared distance.

HAC: Start with every point as its own cluster. Merge the two closest. Repeat until one cluster. Result = dendrogram you can cut at any height. Single-linkage uses min distance between clusters; complete uses max; average uses mean; Ward minimizes variance increase.

DBSCAN: Define ε-neighborhood and min_samples. Points with enough neighbors = "core." Reachable core points form clusters. Handles arbitrary shapes + finds noise points.

PCA: Find directions (eigenvectors of covariance) along which data varies most. Project onto top k → dimensionality reduction.
🔑 Key Terms
Centroid μₖ
Mean of cluster k. K-Means moves these.
Inertia
Σ ‖xᵢ−μ_{c(i)}‖² — K-Means objective.
Elbow Method
Plot inertia vs K, find the bend.
Silhouette
s = (b−a)/max(a,b). +1 good, 0 ambiguous, −1 wrong cluster.
Linkage
How to measure distance between clusters in HAC.
Dendrogram
Binary tree of merges. Cut at any height = clustering.
Core Point (DBSCAN)
Point with ≥ min_samples in ε-radius.
Eigenvector
Direction in PCA. Eigenvalue = variance along it.
Explained Variance
λᵢ / Σλⱼ. Fraction of total variance captured.

  • Running K-Means on concentric circles. K-Means splits by Euclidean distance → cuts the rings in half rather than separating inner/outer.
    For ring/moon shapes, use DBSCAN or spectral clustering.
  • Forgetting to standardize features before K-Means / PCA. A feature in the thousands dominates one in the 0–1 range.
    Always StandardScaler before distance-based methods.
  • Wrong linkage update formula. For single-linkage merge of {A,B}: d({A,B}, C) = min(d(A,C), d(B,C)). Many use max.
    Single → min. Complete → max. Average → mean. Memorize which is which.
  • Confusing PCA components with original features. PC1 is a combination of all features, not one feature.
    Each PC = weighted sum of original features. Loadings tell you the weights.
  • Sorting eigenvectors by eigenvalue magnitude wrong way. PC1 has the LARGEST eigenvalue.
    Sort descending: largest λ first. That's the direction of most variance.
  • Treating K-Means as deterministic. Random init means different runs give different results.
    Use K-Means++ init and run multiple times — sklearn does this by default (n_init).
⚡ Edge case — Single-link "chaining" Single-linkage can connect two distant clusters through a chain of intermediate points. Sometimes useful (elongated clusters), sometimes a bug (sensitive to outliers).
⚡ Edge case — DBSCAN border points Border points are within ε of a core but aren't core themselves. They belong to a cluster but can't extend it.
🧠 Memory hook "K-Means averages. HAC merges. DBSCAN densifies. PCA rotates."

Theory: SVM · Bias-Variance · Regularization · Metrics
The fundamentals that appear in every exam.
TL;DR: Bias = wrong model. Variance = too sensitive to data. L1 = sparse weights. L2 = small weights. SVM = max margin. Cross-entropy for classification, MSE for regression.
💡 Bias-Variance in One Picture
Total error = Bias² + Variance + irreducible noise. High bias = underfitting (model too simple). High variance = overfitting (model memorizes noise). More data reduces variance but not bias. More model complexity reduces bias but increases variance. The sweet spot is found via cross-validation.
💡 SVM, Regularization, Loss Functions
SVM: Maximize the margin 2/‖w‖ between classes. Only support vectors (points on the margin) determine the hyperplane. The kernel trick maps to higher dimensions via K(xᵢ,xⱼ) = φ(xᵢ)·φ(xⱼ) without computing φ explicitly.

L1 vs L2: L1 penalty Σ|wⱼ| → diamond constraint → solution often hits axis corner → some weights become exactly 0 (sparsity). L2 penalty Σwⱼ² → spherical constraint → smooth shrinkage to small but non-zero weights.

Cross-entropy: −Σ yᵢ log ŷᵢ. Gives steep gradient when confident-wrong. Used with sigmoid (binary) or softmax (multi-class) output.
🔑 Key Terms
Bias
Systematic error from oversimplified model.
Variance
Sensitivity to training data noise.
Overfitting
Low train error, high test error.
Underfitting
High train AND test error.
L1 / Lasso
Σ|w|. Produces sparse weights → feature selection.
L2 / Ridge
Σw². Shrinks all weights smoothly.
Support Vector
Point exactly on margin boundary in SVM.

Precision

Recall
TP/(TP+FN). Of real positive, how many caught?
F1 Score

  • Confusing precision and recall. Precision = "of what I PREDICTED positive." Recall = "of what's ACTUALLY positive."
    Precision denominator = TP+FP. Recall denominator = TP+FN.
  • Reporting accuracy on imbalanced data. 95% accuracy on 95/5 split is meaningless (predict majority class).
    Use F1, AUC-ROC, or balanced accuracy for imbalanced classes.
  • Saying L1 always beats L2. L1 helps when you suspect many features are irrelevant. L2 is usually a safer default.
    L1 = feature selection. L2 = general regularization. Elastic Net = both.
  • Believing SVM is "all points matter." Only support vectors (boundary points) determine the hyperplane.
    Far-from-margin points have zero gradient — removing them changes nothing.
  • Forgetting kernel trick = implicit mapping. You don't compute φ(x). You only evaluate K(xᵢ, xⱼ).
    RBF kernel = exp(−γ‖xᵢ−xⱼ‖²). Maps to ∞-dim space without computing it.
  • Wrong sign on bias-variance trade-off. More complexity → less bias, more variance. The opposite for both.
    Simple model = high bias, low variance. Complex = low bias, high variance.
  • Using cross-entropy with sigmoid output for multi-class. Sigmoid is binary; softmax is multi-class.
    Binary → sigmoid + binary cross-entropy. Multi-class → softmax + categorical cross-entropy.
⚡ Edge case — Training accuracy increases, validation accuracy decreases Classic overfitting. Use dropout, L2, data augmentation, or early stopping.
⚡ Edge case — Both training AND validation accuracy stagnate Underfitting. Need more model capacity, less regularization, or better features.
⚡ Edge case — Curse of dimensionality In high dimensions, all points are roughly equidistant. KNN, K-Means, anything Euclidean-based degrades. Reduce dimensions first (PCA) or use methods robust to it (tree-based).
🧠 Memory hook "Bias = wrong shape. Variance = wiggly fit. L1 sparses. L2 shrinks. SVM widens. Cross-entropy punishes confidence."

Exam-Day Tips
Final tactical advice for the test itself.
⏱️ Strategy
1. Skim first. Read every question before answering. Identify low-hanging fruit.
2. Manual calculations: show every step. Partial credit comes from showing you understood the process, even if the final number is wrong.
3. Sanity check arithmetic. Quick check: does Q-value go up or down after a positive reward? Does conv output get smaller after pooling?
4. If stuck on a formula, derive from the concept. Don't remember the LSTM gate? Remember it has 3 sigmoids gating an input/cell/output.
5. Manage time: if a question takes >1.5× expected, move on. Come back later.
  • Spending too long on one question. Easy points elsewhere are worth more than partial credit on hard ones.
    Budget: total minutes / questions. Hard cap per problem.
  • Skipping the "explain in plain English" part. Most rubrics give 30-40% for the intuition.
