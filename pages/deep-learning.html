What is Deep Learning?

Definition

Deep Learning = neural networks with several hidden layers ("depth" = many stacked layers).

Unlike a shallow network (1 hidden layer), deep networks learn hierarchical features automatically:

Early LayersEdges, blobs, low-level features
Middle LayersShapes, textures, parts
Deep LayersEyes, faces, objects (high-level)

Part 2

The Vanishing Gradient Problem

The deeper the network, the worse it gets — gradients shrink exponentially as they travel back through layers.

The Cause

Recall the hidden-unit error signal:

Hidden delta
\[\delta_j = o_j(1-o_j)\sum_k \delta_k w_{kj}\]

The factor \(o_j(1-o_j)\) is the derivative of the logistic function:

Sigmoid Derivative
\[\frac{d}{dx}\sigma(x) = \sigma(x)(1-\sigma(x))\]

Maximum value = 0.25 (at x=0). Always in [0, 0.25].

Going back L layers, error scaled by up to 0.25L!

Part 3

ReLU: The Fix

Rectified Linear Unit

ReLU
\[f(x) = \max(0, x)\]
Derivative
\[f'(x) = \begin{cases} 1 & x > 0 \\ 0 & x \leq 0 \end{cases}\]

Why ReLU Fixes Vanishing Gradients

PropertyBenefit
Gradient = 1 for active nodesError propagates back without downscaling
Faster computeJust a max — no exponentials
Non-linearThe kink at 0 is enough to enable non-linear function approximation

Logistic dominated NNs for decades. ReLU (and variants) is now the default in deep networks.

Sigmoid

Max gradient: 0.25 → shrinks fast

ReLU

Gradient: 0 or 1 → no shrinkage!

Dying ReLU Problem

ReLU outputs 0 for half its domain (x ≤ 0). On average, ~half the neurons may be inactive.

If a neuron outputs 0 for ALL training examples in an iteration, it can never recover — it's dead. Lost capacity.

Part 4

Preventing Overfitting

The Problem

Deep nets have millions of weights. With that capacity, severe risk of overfitting — memorizing training data instead of generalizing.

Typical curve: Training error keeps dropping, but validation error eventually rises as the model memorizes noise.

Early Stopping

Monitor validation error during training; stop when it stops improving. Keep the model at the lowest validation point.

Method 1: Stop on First Increase

Stop the moment validation error goes up compared to the previous epoch.

Pro: Simple
Con: No tolerance for natural fluctuations — noisy curves may trigger premature stopping

Method 2: Patience Parameter

Patience p
\[\text{Stop if } p \text{ consecutive epochs fail to beat } E^* \text{ (best so far)}\]

Patience makes early stopping robust to fluctuations. The standard choice in modern training.

Part 5

Dropout

Randomly deactivate neurons during training to prevent co-adaptation and force robust feature learning.

How Dropout Works

  1. For each training example, randomly drop (zero out) neurons with probability p (e.g., 0.5)
  2. Run forward pass and backprop only on the surviving sub-network
  3. Dropped neurons output 0; their weights don't update for this example
  4. Repeat with a fresh random mask for the next example
  5. At test time: use the entire network (no dropout)
Dropout probability
\[\text{Keep each neuron with probability } (1-p)\]

Why Dropout Prevents Overfitting

EffectWhy It Prevents Overfitting
Smaller sub-networksLess complex → less prone to overfit
Different network per exampleEffectively trains an ensemble of many networks
Same input, different masksNetwork can't memorize specific examples
StabilityOutput becomes robust to small input perturbations

Regularization

Regularization = any technique that modifies the learning process to prevent overfitting, typically by encouraging the model to be stable under small input changes.

Early Stopping

Stop training before memorization sets in. Monitor validation error. Keep best model.

Method: Patience

Dropout

Randomly remove neurons during training. Full network at test time. Acts as ensemble.

Standard technique

Weight Decay (L1/L2)

Add penalty for large weights to the loss function. Keeps weights small and model simple.

\(E_{reg} = E + \lambda\sum w_i^2\)
