Part 1

Biological Inspiration

Neural networks mimic how biological brains process information through connected neurons.

Biological Neuron

🌿
Dendrites — receive signals from other neurons
⚡
Axon — sends signal (if it fires)
🔗
Synapse — connection between neurons; has a strength
📚
Learning = changing the strength of synaptic connections

Artificial Neuron (Perceptron)

📊
Inputs xi ≡ dendrites (features)
⚖️
Weights wi ≡ synaptic strengths
🔢
Net input = weighted sum of inputs
🎯
Output y = fire (1) or don't fire (0)

A stronger synapse = a larger weight. Brain learning = changing the strength of synaptic connections.

Part 2

The Perceptron

Net Input Formula

The perceptron computes a weighted sum of its inputs:

Net Input
\[\text{net} = \sum_i w_i \cdot x_i = w_1 x_1 + w_2 x_2 + \cdots\]

Step Function Output

Output
\[y = \begin{cases} 1 & \text{if net} \geq T \text{ (fires)}\\ 0 & \text{if net} < T \text{ (doesn't fire)} \end{cases}\]

Where T is the threshold. The threshold can be absorbed as a bias: \(b = w_0\) with \(x_0 = 1\).

With Bias
\[\text{Output} = \frac{1}{1+e^{-(\sum_i w_i x_i)}}\]
Interactive Perceptron
x₁ (input 1) 1.0
x₂ (input 2) 2.0
w₁ (weight 1) 0.5
w₂ (weight 2) 0.8
T (threshold) 1.0

Part 3

Perceptron Learning Rule

Weight Update Rule

Weight Update
\[w_i \leftarrow w_i + \eta(y - \hat{y})x_i\]
Threshold Update
\[T \leftarrow T - \eta(y - \hat{y})\]

Where η = learning rate, y = correct output, ŷ = model's prediction, xi = input value.

When Does It Update?

SituationAction
Correct outputDo nothing (y−ŷ = 0)
Predicted 1, should be 0Lower weights on active inputs
Predicted 0, should be 1Raise weights on active inputs

Learning Rate η

η ValueEffect
Typical rangeChosen between 0 and 1
Too smallVery slow learning (too many iterations)
Too largeOscillations; may never converge
Just rightSmooth convergence in reasonable epochs

Convergence Theorems

Convergence Theorem: If data is linearly separable, the Perceptron algorithm will eventually converge.

Cycling Theorem: If data is NOT linearly separable, the algorithm will cycle forever (no solution exists).

Practical fix: Monitor for repeated weight+threshold combinations to detect cycling and guarantee termination.

Part 4

Activation Functions

Non-linear activations give neural networks their expressive power. Without them, stacking linear layers = one linear layer.

Logistic (Sigmoid)

\[\sigma(x) = \frac{1}{1+e^{-x}}\]

Range: (0, 1) — outputs always between 0 and 1.

Derivative: \(\sigma'(x) = \sigma(x)(1-\sigma(x))\)

Max derivative: 0.25 (at x=0)

Problem: Derivative always ≤ 0.25 → vanishing gradients in deep nets!

Key use: Binary classification output layer, gates in LSTMs.

ReLU (Rectified Linear Unit)

\[f(x) = \max(0, x)\]

Range: [0, ∞)

Derivative: 1 if x > 0, else 0

Why it works: Gradient = 1 for active neurons — no downscaling! Fixes vanishing gradients.

Dying ReLU: If input is always negative, neuron permanently outputs 0 and stops learning.

ReLU is non-linear — it behaves differently for positive vs. negative inputs!

Leaky ReLU

\[f(x) = \begin{cases} x & x > 0 \\ \alpha x & x \leq 0 \end{cases}\]

With α = small constant (e.g., 0.01)

Range: (-∞, ∞)

Fixes Dying ReLU: Negative inputs get a small non-zero gradient (αx), so dead neurons can recover!

VariantNegative slope
ReLU0 (flat)
Leaky ReLUFixed α
PReLULearned α

Tanh (Hyperbolic Tangent)

\[\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}\]

Range: (-1, 1) — zero-centered (advantage over sigmoid!)

Derivative: \(1 - \tanh^2(x)\), max = 1 at x=0

Zero-centered helps gradient descent — positive and negative signals. Still saturates, but stronger gradients than sigmoid.

Step Function

\[y = \begin{cases} 1 & \text{if net} \geq T \\ 0 & \text{otherwise} \end{cases}\]

Range: {0, 1}

Derivative: 0 everywhere (except undefined at T)

Problem: Non-differentiable — cannot use gradient descent! Used only in original perceptron, not in modern NNs.

This is why we need smooth activations (sigmoid, ReLU) for backpropagation to work.

Part 5

Multi-Layer Networks

Stacking layers with non-linear activations allows learning complex, non-linear decision boundaries.

Why Multi-Layer?

Perceptron limit: Cannot learn XOR — no single line can separate the 4 XOR examples.

Multi-layer fix: Stack layers + non-linear activation → can learn ANY function!

Why Linear Layers Alone Aren't Enough

Linear stacking = still linear
\[O = (w_5w_1 + w_6w_3)x_1 + (w_5w_2 + w_6w_4)x_2\]

This is still a linear combination of x₁ and x₂ — equivalent to a single perceptron. A non-linear activation is required.

Representational Power

ArchitectureCan Approximate
2-layer (1 hidden + output)Any Boolean function
2-layerAny bounded continuous function
3-layer (2 hidden + output)Any function (to any ε)

Universal Approximation Theorem: A single hidden layer with enough neurons can approximate any continuous function!

Hidden Layers As Feature Detectors

Layer 1Detects edges and simple patterns

Layer NHigh-level concepts (faces, objects)

Hidden nodes can be viewed as new features constructed from the given features — they transform the input space into a learned representation.

Part 6

Backpropagation

Backprop is the algorithm that learns weights by computing how much each weight contributes to the error, then adjusting.

Error Function

Total Squared Error
\[E(W) = \sum_{d \in D} \sum_{k \in K} (t_{kd} - o_{kd})^2\]

Where D = training set, K = output units, tkd = correct, okd = model output.

Weight Update (Gradient Descent)

Backprop Update
\[\Delta w_{ji} = \eta \delta_j o_i\]

Error Signal δ

Delta for Output Unit
\[\delta_j = o_j(1-o_j)(t_j - o_j)\]
Delta for Hidden Unit
\[\delta_j = o_j(1-o_j)\sum_k \delta_k w_{kj}\]

Hidden unit error is propagated back from downstream units via the chain rule.

Logistic Derivative Identity

Key Identity
\[\frac{d\,\text{logistic}(x)}{dx} = \text{logistic}(x) \cdot (1 - \text{logistic}(x))\]

This is what enables efficient computation of δj!

Three-Step Backprop Process

1
Forward Pass
Compute output of every node from input to output layer. Get final predictions.
2
Compute Output δ
Calculate δj = oj(1−oj)(tj−oj) for each output unit.
3
Propagate Back
Compute hidden δ, update all weights: Δwji = η δj oi
Worked Example — δ Computation
oj (output) 0.20
tj (target) 1.0

Softmax + Cross-Entropy δ

Simplified δ for Softmax output
\[\delta_j = \begin{cases} 1 - o_j & \text{if } j \text{ is correct class} \\ -o_j & \text{otherwise} \end{cases}\]

Much cleaner than sigmoid cross-entropy — no o(1-o) saturation factor!

\[P(\text{class}_k) = \frac{e^{z_k}}{\sum_j e^{z_j}}\]

Converts raw scores to probabilities summing to 1.
