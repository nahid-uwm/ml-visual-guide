Convolutional Neural Networks

Convolution, pooling, weight sharing, stride, padding — and why CNNs dominate image tasks.

Part 1

Why Not Dense Layers on Images?

Problems with Dense Layers

❌ Too many weights — a 1000×1000 image × 1000 hidden nodes = 10⁹ weights!
❌ No translation equivariance — dog detector in top-left won't find a dog at bottom-right
❌ No spatial structure — pixels treated as independent features; neighbors ignored
❌ Overfitting risk — billions of parameters for even modest images

CNN Solutions

✅ Local receptive fields — each neuron connects only to a small patch
✅ Weight sharing — the same filter applied everywhere → far fewer parameters
✅ Translation equivariance — pattern detected anywhere in the input
✅ Pooling — spatial invariance + dimensionality reduction

CNN = deep learning architecture designed for grid-like input (images, time series, video).

Part 2

Convolution Layer

A small filter (kernel) slides over the input, computing a dot product at each position to produce a feature map.

Key Terms

TermMeaning
Filter / KernelSmall weight window slid across input
Receptive FieldRegion of the input one neuron "sees"
Feature MapLayer of activations from applying one filter
ConvolveSliding the same function (filter) across the input
StrideHow far the filter window moves between applications

Weight Sharing — Two Big Wins

Fewer parameters: A 6×6 input fully connected to 4×4 hidden = 36×16 = 576 weights. With a 3×3 shared filter: only 9 weights.

Translation equivariance: Whatever pattern the filter learns to detect, it detects it anywhere in the input.

Multiple Filters

One filter detects one pattern. Real tasks need many patterns → use several filters in parallel (e.g. 32, 64, 128). Each filter produces its own feature map.

Feature map depth = number of filters. Connecting multiple feature maps: treat each map as a channel (like RGB).

Part 3

Stride & Padding

Output Size Formula

Conv Output Size
\[\text{Out} = \left\lfloor\frac{W - F + 2P}{S}\right\rfloor + 1\]

W = input width, F = filter size, P = padding, S = stride

PaddingBehavior
ValidNo padding. Skip edge positions. Output shrinks.
SamePad edges with zeros. Output same size as input (stride=1).

Quick example: 6×6 input, 3×3 filter, valid padding, stride 1:
(6−3)/1 + 1 = 4×4 output

Part 4

Pooling Layer

Downsamples feature maps — passes only a summary of each local region. Adds translation invariance.

Why Pool?

📉 Reduces computation in later layers
🎯 Translation invariance — abstracts away exact locations
⚡ No learnable weights — just aggregation
TypeOperation
MaxPoolingOutput = max of values in window (most common)
AvgPoolingOutput = average of values in window

Standard practice: 2×2 window with non-overlapping stride (= 2), valid padding.

Key distinction: Equivariance (convolution): pattern at new location → feature map shifts. Invariance (pooling): pattern at new location → pooled output similar.

2×2 MaxPool Quiz Answer

After applying 2×2 MaxPooling, there are 10 resulting nodes. How many nodes were in the original? A 2×2 pool reduces spatial area by factor 4 (2×2=4 pixels→1 output). So original had 10×4 = 40 nodes.

Transfer Learning & Advanced Topics

Transfer Learning

Training a CNN from scratch needs 10,000s of labeled images for each new task. Most of that effort is spent re-learning generic features (edges, shapes, textures).

Transfer Learning = reuse knowledge learned on one task for another.

Workflow

1Take CNN pretrained on huge dataset (e.g. ImageNet)
2Freeze the early conv layers (generic features already useful)
3Replace final dense+softmax layers with new ones for your task
4Fine-tune the new layers on your small task-specific dataset

Popular pretrained CNNs: VGG-16, ResNet-50, AlexNet — available via keras.applications.

Data Augmentation

Artificially expand the training set with realistic transformed copies:

🔄 Flip, rotate, crop, translate, zoom

🌈 Change contrast / brightness

📐 Any combination of the above

🎯 More robust learning
