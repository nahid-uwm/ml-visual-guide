Q1
k-Nearest Neighbor Prediction. A test example has these distances to seven training examples:
Triangles at distances 2, 8, 9, 10; Squares at distances 3, 3, 4.
(i) What does 1-NN, 3-NN, and 7-NN predict? (ii) Which prediction is likely correct, and why?
⌄
📐 Recreated Diagram
Test point in the middle, with dashed lines showing distance to each of the 7 training examples (4 triangles + 3 squares).
📐 Step-by-Step (sort all distances)
Sorted distances (closest first):
  1. d = 2 → Triangle
  2. d = 3 → Square
  3. d = 3 → Square
  4. d = 4 → Square
  5. d = 8 → Triangle
  6. d = 9 → Triangle
  7. d = 10 → Triangle
✅ Answer (i) — Predictions
  • 1-NN: the single closest neighbor is at d=2 (Triangle) → predict Triangle.
  • 3-NN: the three closest are d=2 (Triangle), d=3 (Square), d=3 (Square) → majority is 2 Squares vs 1 Triangle → predict Square.
  • 7-NN: all 7 examples vote → 4 Triangles vs 3 Squares → predict Triangle.
✅ Answer (ii) — Which is likely correct, and why?
3-NN is most likely correct. The single closest example (1-NN at d=2) may be noise — relying on it alone is risky. 7-NN drags in faraway examples (d=8, 9, 10) which are not local evidence. 3-NN balances locality and noise resistance — it looks at the immediate neighborhood (a small Square cluster around the test point), giving Square as the local-majority class.
💡 Plain English
k controls a bias/variance trade-off: small k = sensitive to noise (overfit), large k = washes out local signal (underfit). Pick k by validation.

Q2
Information Gain. For 4 training examples with binary features F1, F2, F3 and binary target Positive/Negative, compute the information gain when splitting by each feature. Which feature would a decision tree pick at the root?
⌄
📋 Training Data
ExampleF1F2F3Target
1TrueTrueTruePositive
2TrueFalseFalseNegative
3TrueFalseTrueNegative
4FalseTrueFalsePositive
📐 Step 1 — Initial entropy
2 Positive + 2 Negative out of 4 → 50/50 → entropy is at its maximum:
H(target) = −(2/4)·log₂(2/4) − (2/4)·log₂(2/4) = 1.0
📐 Step 2 — Information Gain for F1
F1 = True (Ex 1, 2, 3): 1 Positive + 2 Negative → entropy = −(1/3)log₂(1/3) − (2/3)log₂(2/3) ≈ 0.92
F1 = False (Ex 4): 1 Positive + 0 Negative → entropy = 0 (pure)
IG(F1) = 1 − [(3/4)(0.92) + (1/4)(0)] = 1 − 0.69 = 0.31
📐 Step 3 — Information Gain for F2
F2 = True (Ex 1, 4): 2 Positive + 0 Negative → entropy = 0 (pure)
F2 = False (Ex 2, 3): 0 Positive + 2 Negative → entropy = 0 (pure)
IG(F2) = 1 − [(2/4)(0) + (2/4)(0)] = 1.0 (maximum possible!)
📐 Step 4 — Information Gain for F3
F3 = True (Ex 1, 3): 1 Positive + 1 Negative → entropy = 1 (worst case, still 50/50)
F3 = False (Ex 2, 4): 1 Positive + 1 Negative → entropy = 1
IG(F3) = 1 − [(2/4)(1) + (2/4)(1)] = 0 (useless!)
✅ Final Answer
IG(F1) = 0.31, IG(F2) = 1.0, IG(F3) = 0.
A decision tree picks the feature with the highest information gain, so it picks F2 as the root split. F2 perfectly separates Positive from Negative — both branches become pure leaves immediately. F3 is completely useless (gain = 0); F1 is helpful but not as decisive as F2.
💡 Plain English
Information gain measures how much a split reduces randomness. A "pure" subset has zero entropy (you know the answer). F2 is the dream feature here — it splits the data into two perfectly pure groups, so all the uncertainty disappears in one step.

Q3
Naïve Bayes Probability Estimation. A binary classification task has features Sky and Temp. (i) Fill in the probability table without smoothing. (ii) Which probabilities make smoothing necessary? (iii) Express P(sky=cloudy, temp=hot | positive).
⌄
📋 Training Data (6 examples)
ExampleSkyTempLabel
1sunnyhotnegative
2cloudycoldnegative
3rainypleasantpositive
4sunnycoldpositive
5cloudypleasantpositive
6rainyhotpositive
4 positives (Ex 3, 4, 5, 6) and 2 negatives (Ex 1, 2).
✅ Answer (i) — Probability Estimates
Probabilitypositivenegative
P(Y)4/62/6
P(sky = sunny | Y)1/41/2
P(sky = cloudy | Y)1/41/2
P(sky = rainy | Y)2/40/2
P(temp = hot | Y)1/41/2
P(temp = cold | Y)1/41/2
P(temp = pleasant | Y)2/40/2
📐 How to read the table
For each cell, count how many training examples of that class have that feature value, then divide by the total examples in that class. Example: P(sky=rainy | positive) — among 4 positives, examples 3 and 6 are rainy → 2/4. P(sky=rainy | negative) — among 2 negatives, none are rainy → 0/2.
✅ Answer (ii) — Zero probabilities (smoothing needed)
The two cells highlighted in red are zero:
  • P(sky = rainy | negative) = 0
  • P(temp = pleasant | negative) = 0
Without smoothing, ANY test example with sky=rainy or temp=pleasant would get P(features | negative) = 0 (zero anywhere in the product kills the whole thing) — so Naïve Bayes would refuse to ever predict "negative" for those examples, even when other features strongly suggest negative. Laplace smoothing adds a small count (+1) to each cell to avoid this collapse.
✅ Answer (iii) — Joint conditional under NB assumption
P(sky=cloudy, temp=hot | positive) = P(sky=cloudy | positive) × P(temp=hot | positive)
Naïve Bayes assumes features are conditionally independent given the target, so the joint conditional factors into a product. Plugging in: (1/4) × (1/4) = 1/16.
💡 Plain English
Naïve Bayes assumes once you know the class, the features don't influence each other. That lets you multiply individual conditional probabilities instead of estimating one giant joint table — which is why NB needs only LINEAR (not exponential) parameters in the number of features.

Q4
Evaluation Metrics. 100 test examples: 60 positive, 40 negative. The classifier correctly classified 50 of the 60 positives and 20 of the 40 negatives. Compute sensitivity, specificity, precision, recall, and accuracy.
⌄
📋 Confusion Matrix
Predicted +Predicted −Row total
Actual +TP = 50FN = 1060
Actual −FP = 20TN = 2040
Column total7030100
📐 Step 1 — Extract TP, FN, FP, TN
  • TP = 50 (positives correctly predicted positive)
  • FN = 10 (60 positives − 50 caught = 10 missed)
  • TN = 20 (negatives correctly predicted negative)
  • FP = 20 (40 negatives − 20 caught = 20 wrongly flagged positive)
✅ Computed Metrics
  • Sensitivity = TP / (TP + FN) = 50 / 60 = 5/6 ≈ 0.833
  • Specificity = TN / (TN + FP) = 20 / 40 = 1/2 = 0.50
  • Precision = TP / (TP + FP) = 50 / 70 = 5/7 ≈ 0.714
  • Recall = TP / (TP + FN) = 50 / 60 = 5/6 ≈ 0.833 (same as sensitivity)
  • Accuracy = (TP + TN) / Total = (50 + 20) / 100 = 0.70
💡 Plain English
Sensitivity (=Recall) answers: of all real positives, how many did we catch? Here, 5 out of 6 → good.
Specificity answers: of all real negatives, how many did we correctly leave alone? Here, only half → the classifier is over-eager to predict positive.
Precision answers: of those we predicted positive, how many were actually positive? Here, 5 out of 7 → reasonable.
Accuracy is overall correctness — 70 out of 100.

Q5
Curves. Label the two axes and draw a typical shape for: (i) a learning curve for accuracy, (ii) an ROC curve.
⌄
📈 (i) Learning Curve
Y-axis: Test Accuracy · X-axis: Number of training examples. Accuracy rises steeply with the first few examples, then flattens — more data eventually stops helping.
📈 (ii) ROC Curve
Y-axis: Sensitivity (TPR) · X-axis: 1 − Specificity (FPR). A good ROC bends toward the top-left corner (perfect = (0, 1)). Diagonal dashed line is random guessing.
💡 Plain English
Learning curve answers "do I need more data?" — if the curve has plateaued, more data won't help; if it's still rising, collect more.
ROC curve answers "how do I tune the decision threshold?" — each point is one threshold. The curve shape summarizes ranking quality; AUC is its single-number summary.

Q6
Inductive Bias. (i) Besides consistency with training data, what is fundamentally necessary for an ML method to predict on previously unseen examples? (ii) Can two methods that perfectly fit the training data make different predictions on test data?
⌄
✅ Answer (i)
Inductive bias is fundamentally necessary. Training data alone is consistent with infinitely many functions — without a preference (a "bias") for some hypotheses over others, the learner has no principled way to pick one extrapolation over another. Inductive bias = the set of assumptions a learner uses to generalize.
✅ Answer (ii)
Yes, because they may have different inductive biases. Two models that fit the training data perfectly can still disagree on unseen examples — they extrapolate according to different assumptions about which patterns are "natural" or "preferred".
🖼️ Visual Intuition
Both curves pass through all four training points perfectly, but predict wildly different values for new inputs — the difference is purely inductive bias (preference for simpler vs more flexible functions).
💡 Plain English
"No free lunch" — without assumptions, no learner can outperform random guessing on average across all problems. Useful learners commit to assumptions about what real-world patterns look like (smoothness, low complexity, locality, etc.).

Q7
Gradient Descent. Identify each symbol in the update rule, and state one convergence criterion.
⌄
📐 The Update Rule
✅ Answer (i) — Symbol Meanings
SymbolMeaning
tᵢThe actual (true) target value of training example i
M_w(dᵢ)The model's predicted value for example i (using current weights w)
(tᵢ − M_w(dᵢ))The prediction error on example i (true minus predicted)
w[j]The weight for feature j (the parameter being updated)
αThe learning rate — how big each update step is
dᵢ[j]The value of feature j in example i
ΣᵢSum over all n training examples — this is batch gradient descent
📐 Intuition for the update direction
  • If the error (tᵢ − M_w(dᵢ)) is positive (model predicted too low), we want to INCREASE weights for features whose dᵢ[j] is positive — that nudges the prediction up next time.
  • If the error is negative (predicted too high), the same product term is negative, pushing the weight down.
  • Multiplying by feature value dᵢ[j] concentrates the update on features that actually played a role in the prediction.
  • α controls step size: too big → diverge or oscillate; too small → slow convergence.
✅ Answer (ii) — Convergence Criterion
Common criteria include:
  • Small change in error from one iteration to the next: |L_t − L_{t−1}| < ε. The algorithm has stopped improving.
  • Small gradient norm: ‖∇L‖ < ε. We're near a stationary point.
  • Maximum iterations reached (safety cap).
  • Validation error stops decreasing (early stopping).

Q8
Short-Answer Round-Up. Five sub-questions on evaluation, feature selection, curse of dimensionality, overfitting prevention, and distance measures.
⌄
✅ (i) Why not tune parameters using test data?
The test set must remain unseen until final evaluation. Tuning on test data leaks information — every choice you make based on test performance fits the test set itself, so the reported number becomes a biased overestimate of true generalization. Use a separate validation set for tuning.
✅ (ii) What features could forward feature selection miss?
Interacting features. Forward selection adds one feature at a time based on individual gain. Two features that are weak alone but powerful together (XOR-like patterns) are never picked because neither one individually improves the score.
✅ (iii) High-dimensional curse — name and most-affected method
The phenomenon is the curse of dimensionality. The method most affected is k-Nearest Neighbor. kNN depends on the meaningfulness of distance, and in high dimensions all pairs of points have similar Euclidean distances → the notion of "nearest" loses discriminative power.
✅ (iv) Preventing overfitting
  • k-Nearest Neighbor: use a larger k (chosen via validation / internal cross-validation). Larger k smooths out noise — 1-NN is the extreme overfitting case.
  • Decision Tree: use pruning — pre-pruning (stop growing when a criterion is met, e.g., min samples per leaf, max depth) or post-pruning (build full tree then trim back nodes whose removal improves validation accuracy, a.k.a. Reduced Error Pruning).
✅ (v) Distance measures
  • Sequence of characters → Edit distance (a.k.a. Levenshtein distance) — counts insertions, deletions, substitutions needed to turn one string into another.
  • Time series data → Dynamic Time Warping (DTW) — aligns two sequences in time (allowing stretches and shifts) before computing distance, robust to phase shifts that would break Euclidean distance.
