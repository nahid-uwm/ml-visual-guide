🧠 Neural Network Architecture & Backprop

Q1 from Sample-Questions PDF
V1
The figure shows a neural network that is not fully-connected. When training this network, errors from which nodes will be used to compute the error at node h₂? Updates on which weights will be influenced by the error at h₂?
⌄
Pink edges = connections involving h₂ · only o₁ receives from h₂ · h₂ connects down to i₁, i₃, i₅
✅ Official Answer
Error from only o₁ will be used to compute the error at h₂. Weights of the edges between (h₂, i₁), (h₂, i₃), (h₂, i₅) will be influenced by the error computed at h₂.
📐 Step-by-Step Reasoning
  1. Identify outgoing edges of h₂. h₂ only connects forward to o₁ (no edge to o₂). So only o₁'s error backpropagates to h₂.
  2. Compute δ_h₂ = δ_o₁ · w(h₂→o₁) · σ'(z_h₂). Standard backprop chain rule — sum only over o₁ since o₂'s gradient cannot reach h₂.
  3. Identify incoming edges to h₂. h₂ receives from i₁, i₃, i₅. These are the weights that need updating.
  4. Weight update for incoming edge: Δw(iⱼ→h₂) = −η · δ_h₂ · iⱼ. Only the 3 incoming weights of h₂ change.
💡 Plain English
Backprop is local: a hidden neuron only "hears" gradient from nodes it sends to, and only updates weights coming into it. h₂ never connected to o₂ in the forward pass, so o₂'s error cannot blame h₂. h₂ talks to o₁ and listens from {i₁, i₃, i₅} — those are the only relevant edges.
🔗 Related Practice Questions
Q-related-1: In a fully-connected 3-layer net with 2 hidden neurons and 2 outputs, what gradients reach hidden neuron h₁? Answer: gradients from BOTH outputs (o₁ and o₂) since h₁ connects to both. Sum: δ_h₁ = σ'(z_h₁)·(δ_o₁·w_{h₁o₁} + δ_o₂·w_{h₁o₂}).
Q-related-2: Why does dropout effectively "prune" connections like this non-fully-connected example during training? Answer: dropout randomly zeros neurons → their outgoing edges effectively don't exist for that pass → backprop ignores them, just like missing edges in this question.
Q-related-3: If h₂ had connections to BOTH o₁ and o₂, would the gradient be larger or smaller? Answer: typically larger in magnitude (sum of two contributions vs one), but sign depends on whether the contributions align. More incoming gradient ≠ better training necessarily.
V2
A neural network is designed for a classification task with 4 classes and 7 features. How many nodes in the output layer? What activation function? How do output values indicate the class?
⌄
✅ Official Answer
4 nodes in the output layer. Activation: softmax. Softmax outputs probabilities for each of the 4 classes — the class with the highest probability is the prediction.
📐 Reasoning
7 features = 7 input nodes (irrelevant to output layer). 4 classes need one output node per class. Softmax: ŷᵢ = e^{zᵢ} / Σⱼ e^{zⱼ}. All outputs sum to 1 → valid probability distribution. Predicted class = argmax over outputs.
🔗 Related Practice Questions
Sub-topic — binary classification: For 2 classes, how many output nodes and what activation? Answer: 1 node with sigmoid (outputs P(class=1); P(class=0)=1−ŷ). Alternatively 2 nodes with softmax — equivalent.
Sub-topic — regression: For predicting a continuous price, how many output nodes and what activation? Answer: 1 node with NO activation (or linear). MSE loss instead of cross-entropy.
Sub-topic — multi-label: A photo can have multiple tags (cat, sunset, beach). What output layer? Answer: 1 node per tag with sigmoid each (independent). NOT softmax — that forces probabilities to sum to 1.
V3
Name the ML method covered in the course: (i) most human-interpretable classification, (ii) no training work, all work at testing, (iii) needs only one pass through training data, (iv) designed specifically for sequential inputs.
⌄
✅ Official Answer
(i) Decision Trees · (ii) K-Nearest Neighbors (kNN) · (iii) Naïve Bayes · (iv) Recurrent Neural Networks (RNN)
📐 Why Each
  1. Decision Tree: outputs a human-readable if-then-else rule chain. You can literally read off the classification logic.
  2. kNN: "lazy learner" — stores all training data at training time, does all distance computation at test time.
  3. Naïve Bayes: probabilities estimated by counting frequencies in one pass. No gradient descent loop.
  4. RNN: hidden state hₜ carries information from past timesteps — the only architecture in the course explicitly designed for sequential data.
🔗 Related Practice Questions
Sub-topic — most accurate on tabular data: Which method? Answer: Random Forest / Gradient Boosting (ensembles of decision trees) — dominates Kaggle.
Sub-topic — best for image data: Which method? Answer: CNN (Convolutional Neural Network) — weight sharing + spatial structure.
Sub-topic — maximum margin separator: Which method? Answer: SVM (Support Vector Machine).
Sub-topic — uses Bayes' theorem + independence assumption: Which method? Answer: Naïve Bayes.
V4
Linear regression predicts a number between −∞ and +∞. Can linear regression be adapted to predict a binary class? If yes, how? If not, why not? (2–3 sentences)
⌄
✅ Official Answer
Yes — this is logistic regression. Linear regression is used to predict the log-odds (log p/(1−p)) which itself ranges over (−∞, +∞). The logistic (sigmoid) function then maps the log-odds back to a probability in (0, 1). Threshold at 0.5 for the binary class.
💡 Key Insight
The link function (sigmoid) is what bridges unbounded linear output to bounded probability. This is exactly why logistic regression is in the "generalized linear models" family — linear regression's wᵀx + b is preserved, only the output transformation changes.
🔗 Related Practice Questions
Sub-topic — multi-class: Can logistic regression handle 4 classes? Answer: yes, via softmax regression (multinomial logistic regression) — one set of weights per class, softmax over outputs.
Sub-topic — non-linear regression: Can linear regression fit a curve? Answer: yes, by adding higher-order features (x², x³, sin x, x₁x₂) — polynomial/basis-expanded regression. Model stays linear in PARAMETERS w even though non-linear in input x.
Sub-topic — non-linear classification: Will logistic regression work on a circular dataset? Answer: no — logistic regression draws a linear boundary. Solution: add F₁², F₂² as features → linear in expanded space, circular in original.

🖼️ CNN — Convolution & Layer Identification

Q1, Q2, Q5 across sample exams
V5
A 4×4 image with one channel. Using a 3×3 convolution filter with stride (1,1) and valid padding: (i) output size? (ii) unique weights in the filter (ignore bias)? (iii) if instead a dense layer with 3 nodes connects to the image, how many weights (ignore bias)? (iv) one advantage of conv over dense.
⌄
✅ Official Answers
(i) 2×2 · (ii) 9 weights · (iii) 16 × 3 = 48 weights · (iv) Translation equivariance (the same feature detector works at every position)
📐 Step-by-Step
  1. Output size formula: ⌊(W − F + 2P)/S⌋ + 1 = ⌊(4 − 3 + 0)/1⌋ + 1 = ⌊1⌋ + 1 = 2. Both dimensions → 2×2.
  2. Filter weights: 3 × 3 × 1 (single channel) = 9 unique weights, reused at every spatial position (weight sharing).
  3. Dense layer weights: input flattened to 16 nodes. Dense layer with 3 nodes → 16 × 3 = 48 weights. No sharing — every pair gets its own weight.
  4. Why conv wins: 9 weights vs 48 = 5× fewer. AND same filter detects an edge wherever it is — translation equivariance — impossible with dense layer.
🔗 Related Practice Questions
Sub-topic — same padding: What padding P keeps output size = input size when F=3, S=1? Answer: P = 1 (one zero on each side). General rule: P = (F−1)/2 for odd F.
Sub-topic — multi-channel input: 32×32×3 RGB image, 64 filters of 3×3. Parameters? Answer: (3·3·3 + 1) × 64 = 28 × 64 = 1,792 (includes bias per filter).
Sub-topic — output after pooling: Following the 2×2 conv output, apply 2×2 max-pool stride 2. Result? Answer: ⌊(2−2)/2⌋ + 1 = 1 → output is 1×1.
Sub-topic — receptive field: After two stacked 3×3 conv layers, what's the receptive field of one output unit? Answer: 5×5 (each layer adds (F−1) to the receptive field).
V6
Name the neural network layer: (i) prevents overfitting, (ii) downsamples to reduce computational load, (iii) outputs classification probabilities, (iv) makes it possible to identify something in one part of the input even though it was trained with examples having it in other parts.
⌄
✅ Official Answers
(i) Dropout layer · (ii) Pooling layer · (iii) Softmax layer · (iv) Convolution layer
📐 Why Each
  1. Dropout — randomly zeros neurons during training, breaks co-adaptation → reduces overfitting.
  2. Pooling — max or average over windows. Halves spatial dimensions (no learnable params).
  3. Softmax — turns logits into probabilities summing to 1 → final classification layer.
  4. Convolution — weight sharing means a feature learned in one image region transfers everywhere → translation equivariance.
⚠️ Common Mistake
Don't say "ReLU prevents overfitting" — ReLU is an activation function, not a regularizer. Don't say "pooling adds translation invariance" alone — convolution gives equivariance, pooling adds the invariance.
🔗 Related Practice Questions
Sub-topic — batch normalization purpose? Answer: stabilizes activations by normalizing per mini-batch → allows higher learning rates, faster training.
Sub-topic — what does ReLU prevent that sigmoid doesn't? Answer: vanishing gradient (ReLU's derivative = 1 for z>0; sigmoid's max derivative = 0.25).
Sub-topic — average pooling vs max pooling? Answer: max keeps strongest activation (sharper features). Average smooths. Global Average Pooling can replace final FC layers entirely.
V7
What enables deep neural networks to directly work with raw features (e.g. pixel values) without needing us to manually engineer features? (2–3 sentences)
⌄
✅ Official Answer
In deep neural networks, feature engineering is part of the learning process. Hidden-layer nodes learn higher-order features over the features of the previous layer, starting from the raw features of the input.
💡 Hierarchy of Features (CNN Example)
Layer 1: edges/colors → Layer 2: textures/corners → Layer 3: object parts (eyes, wheels) → Layer 4: full objects. The network discovers this hierarchy automatically by minimizing the loss — no SIFT/HOG/manual descriptors needed.
🔗 Related Practice Questions
Sub-topic — why was feature engineering needed pre-deep-learning? Answer: classical methods (SVM, logistic reg, NB) cannot learn from raw pixels — they need numerical features. Researchers had to hand-design SIFT, HOG, Gabor filters etc.
Sub-topic — what risks come with auto-feature learning? Answer: needs huge data + compute; learned features hard to interpret; can pick up spurious correlations in training data.
V8
Two activation function plots are shown: (i) piecewise-linear, zero for x<0, identity for x≥0, range [0,10] over x∈[-10,10]; (ii) S-shaped curve, asymptotes at 0 and 1. Name both. Which is preferred in deep networks and why?
⌄
✅ Official Answer
(i) ReLU activation function · (ii) Sigmoid (logistic) activation function. ReLU is preferred in deep networks because sigmoid suffers from the vanishing gradient problem — sigmoid's derivative ≤ 0.25, so over many layers the backpropagated errors shrink toward zero from repeated multiplications.
📐 Vanishing Gradient Math
  1. Sigmoid derivative: σ'(z) = σ(z)(1−σ(z)). Maximum = 0.25 (at z=0).
  2. Chain rule: ∂L/∂w_early = product of σ'(z) at every layer between output and early layer.
  3. After 10 sigmoid layers: gradient ≤ 0.25¹⁰ ≈ 9.5 × 10⁻⁷ → effectively zero.
  4. ReLU: derivative = 1 for z>0 → no shrinkage. Early layers receive proper gradient.
🔗 Related Practice Questions
Sub-topic — dying ReLU: What's the downside of ReLU and how is it fixed? Answer: neurons with always-negative pre-activation output 0 forever → 0 gradient → dead. Fix: Leaky ReLU (α·x for x<0) or PReLU (learned α).
Sub-topic — softmax purpose: Why is softmax used at the output (not sigmoid) for multi-class? Answer: softmax gives mutually exclusive probabilities summing to 1. Sigmoid would give independent probabilities each in (0,1) — not normalized.
Sub-topic — Tanh: Why tanh sometimes better than sigmoid? Answer: tanh is zero-centered (range (−1, 1)), so gradients of mean zero rather than always positive — helps optimization. But still saturates → not great for deep nets.
V9
Describe in 1–2 sentences the purpose of each: (i) convolutional layer, (ii) pooling layer, (iii) dropout layer.
⌄
✅ Official Answer
(i) Convolutional layer — achieves translation equivariance (same thing detected anywhere in input) and reduces weights via weight sharing.
(ii) Pooling layer — achieves translation invariance and reduces computational load.
(iii) Dropout layer — prevents overfitting.
⚠️ Equivariance vs Invariance — Common Confusion
Convolution is equivariant: shift input by 1 pixel → output shifts by 1 pixel. Pooling adds invariance: within a window, exact position no longer matters. Together they make CNNs robust to small shifts.

🔄 RNN — Unrolling & Backprop Through Time

Q6 from Sample-Final-Exam2
V10
Figure 1: an RNN with weights W_hx, W_hh, W_yh. Figure 2: unrolled over 3 timesteps with connections A–H. (i) What weights are on A–H? (ii) Error from output Y₁ backpropagates through which connections (A–H)?
⌄
Red highlighted: edges that backpropagate Y₁'s error (G → B → D → A)
✅ Official Answer
(i) Weights for A, B, C = W_hx · Weights for D, E = W_hh · Weights for F, G, H = W_yh
(ii) Error from Y₁ backpropagates through G, B, D, A.
📐 Step-by-Step Reasoning
  1. Weight assignment: RNN shares weights across time. Input→hidden uses W_hx everywhere (A, B, C). Hidden→hidden uses W_hh (D, E). Hidden→output uses W_yh (F, G, H).
  2. Y₁ backprop path: Y₁ comes from h₁ via G (hidden→output at t=1).
  3. h₁ depends on x₁ via B AND on h₀ via D.
  4. h₀ depends on x₀ via A.
  5. So gradient flows: G → B (and through h₁ to D → A).
💡 Why "Backprop Through Time" (BPTT)
Even though the unrolled network looks deeper, the same shared weight W_hh gets gradient contributions from EVERY timestep that uses it. During SGD update, you sum all contributions: ∂L/∂W_hh = Σₜ (gradient at timestep t).
🔗 Related Practice Questions
Sub-topic — Y₂ backprop path: Through which edges does Y₂'s error propagate? Answer: H, C, E, B, D, A. Y₂ comes from h₂ via H; h₂ from x₂ (C) and h₁ (E); h₁ from x₁ (B) and h₀ (D); h₀ from x₀ (A).
Sub-topic — vanishing gradient in RNN: Why does this BPTT path cause vanishing gradient? Answer: gradient multiplied by W_hh and tanh' at every backward step. After many timesteps, product → 0.
Sub-topic — LSTM fix: How does LSTM mitigate the vanishing gradient? Answer: cell state Cₜ flows through unchanged (no multiplication by W) — gradient highway. Gates control modification multiplicatively.
Sub-topic — parameter count: If hidden dim = 100, input dim = 50, output dim = 10, total RNN parameters (no bias)? Answer: W_hx (50×100) + W_hh (100×100) + W_yh (100×10) = 5000 + 10000 + 1000 = 16,000.

🎯 Attention & Self-Attention

Q9 from Sample Final Exam (PDF 4)
V11
For "I ate apple", new representation ate' is built by self-attention using four functions f, g, h, i:
ate' = f(I)·g(h(ate), i(I)) + f(ate)·g(h(ate), i(ate)) + f(apple)·g(h(ate), i(apple))Which function is: (i) compatibility, (ii) value, (iii) query, (iv) key?
⌄
✅ Official Answer
(i) Compatibility = g · (ii) Value = f · (iii) Query = h · (iv) Key = i
📐 How to Decode the Formula
  1. What gets multiplied by what? f(word) is being scaled by a scalar weight g(...,...). That weight is the attention coefficient → g is the compatibility (Q·K) function. f is the value (V).
  2. g takes two inputs: h(ate) — the SAME word every time (this is the query, asking the question), and i(other_word) — different each time (the key being matched against). So h = query, i = key.
  3. Cross-check: in standard attention, output_for_ate = Σ_w softmax(Q_ate · K_w) · V_w. Matches: g(h(ate), i(w)) = compatibility(Q_ate, K_w); f(w) = V_w.
💡 Memory Hook
The query is whose representation you're computing (h is applied to "ate" three times — it's the question being asked). Keys are what each word advertises (i is applied to all three different words). Values are what gets returned (f, multiplied by the weights).
🔗 Related Practice Questions
Sub-topic — scaled dot-product attention: What does √dₖ do in Attention(Q,K,V) = softmax(QKᵀ/√dₖ)V? Answer: prevents the dot product from growing too large with high dₖ, which would saturate softmax (near-zero gradients).
Sub-topic — multi-head attention purpose: Why use h parallel heads instead of one big head? Answer: different heads specialize in different relationship types (syntax, coreference, position). Same param count as single head with d-dim Q/K/V.
Sub-topic — positional encoding: Why is it needed? Answer: attention itself is permutation-invariant — "dog bites man" and "man bites dog" produce same attention weights without position info. PE injects order.
Sub-topic — masked self-attention: Used in encoder or decoder, and why? Answer: decoder (GPT-style). Mask prevents token t from attending to t+1, t+2, … so generation is causal.

🎨 GAN — Components & Training

Q6 from Sample Final Exam (PDFs 2 & 4)
V12
A GAN diagram shows components A, B, C, D. (i) Name them. (ii) Which component is being trained in this figure? (iii) Which component is used after the entire network has been trained? (iv) What is a GAN used for?
⌄
✅ Official Answers
(i) A = Random vector · B = Generator · C = Discriminator · D = Training data
(ii) The Discriminator is being trained (the figure shows it receiving both fake and real samples to classify).
(iii) After training, only the Generator is used (to produce new samples from random z).
(iv) To generate realistic novel examples (e.g. images).
📐 GAN Training Cycle
  1. Discriminator phase (this figure): sample real x from D, fake G(z) from B. Train C to output 1 for real, 0 for fake.
  2. Generator phase: freeze C. Sample z, generate G(z). Train B to make C output 1 on G(z) (fool the discriminator).
  3. Alternate until equilibrium — both phases happen many times per epoch.
  4. At inference: discard C. Sample z ~ N(0,I), output G(z). That's the new image.
🔗 Related Practice Questions
Sub-topic — mode collapse: What is it? Answer: G finds one output (or a few) that fools D and produces only those — diversity is lost.
Sub-topic — Conditional GAN: How does cGAN let you say "generate a cat"? Answer: pass class label y as additional input to BOTH G and D. G learns to produce class-y samples; D learns "real/fake given y".
Sub-topic — Diffusion vs GAN: Why are diffusion models more stable? Answer: each step is supervised regression (predict noise). No adversarial game → no mode collapse, no oscillation. Trade-off: slow sampling.
Sub-topic — VAE vs GAN output quality: Answer: VAE outputs are blurrier (MSE averages plausible reconstructions). GAN outputs are sharper (adversarial signal prevents blur).

🔁 Autoencoder Identification

Q6 from Sample-Questions PDF
V13
A neural network has hourglass shape: 5 input nodes (values 1.2, 2.3, 1.9, 0.5, 1.6) at the bottom, narrowing through hidden layers to a single bottleneck, then widening to 5 output nodes (same values). Identify the type and purpose.
⌄
Same values at input and output · narrow bottleneck in middle
✅ Official Answer
This is an Autoencoder. It is used for representation learning — deriving new compact features from the given features by forcing reconstruction through a narrow bottleneck.
📐 Why It Works
The bottleneck (1 node here) cannot pass all 5 input values through unchanged. The encoder must learn to compress the input into the most informative single number; the decoder must learn to reconstruct from it. The bottleneck representation is the learned feature.
🔗 Related Practice Questions
Sub-topic — denoising autoencoder: How does it differ? Answer: input has noise added during training, but target is the clean version. Network learns robust features that ignore noise.
Sub-topic — VAE vs AE: What's the key difference? Answer: VAE encoder outputs (μ, σ) defining a distribution, plus KL regularizer pulls latent toward N(0,I). AE encodes to a point. Only VAE can SAMPLE new examples.
Sub-topic — anomaly detection: How can AE be used? Answer: train on normal data only. At test, large reconstruction error = anomaly (input doesn't match learned distribution).
Sub-topic — why blurry outputs? Answer: MSE loss penalizes pixel-by-pixel error → averages over plausible reconstructions → smooth/blurry output. GAN avoids this via adversarial signal.

🎮 Reinforcement Learning & Q-Learning

Q7 (Sample-Questions) & Q5 (Final-Exam PDFs)
V14
The Q-learning update equation is:
Q_{k+1}(s,a) = (1−α)·Q_k(s,a) + α·(R(s,a,s′) + γ·max_{a′} Q_k(s′,a′))What are s, a, s′, α, R(s,a,s′), and Q_{k+1}(s,a)?
⌄
✅ Official Answer
sCurrent state
aAction taken from the current state
s′Next state obtained by taking action a from s
αLearning rate
R(s,a,s′)Reward received when going from state s to s′ after taking action a
Q_{k+1}(s,a)Estimate in the (k+1)-th iteration of all future rewards obtained by taking action a from state s
γDiscount factor (not asked but key) — weight on future rewards
💡 Q-Value Definition (Q5 in multiple PDFs)
Q-value Q(S, A) for a state S and action A is the expected total future reward if action A is taken from state S. Once Q is learned, the optimal policy is π*(s) = argmax_a Q(s, a).
🔗 Related Practice Questions
Sub-topic — numerical computation: Q(s,a)=3, R=5, max Q(s′,a′)=10, α=0.1, γ=0.9. New Q(s,a)? Answer: (1−0.1)·3 + 0.1·(5 + 0.9·10) = 2.7 + 0.1·14 = 2.7 + 1.4 = 4.1.
Sub-topic — γ = 0: What does the agent care about? Answer: only the immediate reward R. Future rewards are ignored — shortsighted policy.
Sub-topic — ε-greedy: Why is it used? Answer: balances exploration vs exploitation. With probability ε pick random action (explore); else argmax_a Q(s,a) (exploit).
Sub-topic — DQN: Why use a neural network for Q instead of a table? Answer: state space too large to enumerate (e.g., Atari pixels). NN generalizes Q to unseen states.
Sub-topic — off-policy vs on-policy: Q-learning is which, and what's the difference from SARSA? Answer: Q-learning is off-policy — uses max over next actions (best possible). SARSA is on-policy — uses Q(s′, a′ actually taken by ε-greedy).

🧮 Perceptron Training Rule

Q7 from all 3 main exam PDFs
V15
The perceptron weight update rule is wᵢ = wᵢ + η(y − y′)xᵢ. (i) What are η, y, y′, xᵢ? (ii) Under what circumstance does the perceptron training algorithm converge?
⌄
✅ Official Answers
(i) η = learning rate · y = correct output · y′ = predicted output · xᵢ = i-th input.
(ii) The perceptron training algorithm converges when the data is linearly separable (perceptron convergence theorem).
📐 Why The Rule Works
  1. If prediction is correct: y − y′ = 0 → no update.
  2. If wrong: y − y′ ≠ 0 → weight changes proportional to xᵢ.
  3. For active inputs (xᵢ ≠ 0), weights move toward the correct decision.
  4. Guaranteed to find separating hyperplane in finite steps IF one exists (Rosenblatt 1958).
🔗 Related Practice Questions
Sub-topic — non-separable data: What happens if data isn't linearly separable? Answer: perceptron oscillates forever, never converges. Need MLP, kernel SVM, or any non-linear method.
Sub-topic — XOR: Why can't perceptron solve XOR? Answer: XOR isn't linearly separable — no straight line separates (0,1)+(1,0) from (0,0)+(1,1).
Sub-topic — bias term: How is the bias updated? Answer: b = b + η(y − y′). Same rule with x₀ = 1 (bias as 0-th input).
Sub-topic — MLP advantage: What makes MLP solve XOR? Answer: hidden layer + non-linear activation. Two layers of linear-then-sigmoid can carve out non-linear regions.

📐 Classical ML — SVM, Logistic Regression, Overfitting

Q8, Q9 across sample exams
V16
Positive (red empty circles) and negative (blue filled circles) examples are shown with 2 features. Three separators A, B, C are drawn. Which is most likely to be learned by SVM and why?
⌄
Three candidate separators · cyan (B) is the maximum-margin choice
✅ Official Answer
Separator B is most likely to be learned by SVM. SVM learns the separator that maximizes the margin between training examples on either side — hence the middle-most line.
📐 Why Not A or C?
  1. A hugs the positive class too closely — tiny margin → poor generalization to new points slightly to the left.
  2. C is a curve. SVM (linear kernel) only learns hyperplanes (straight lines in 2D). A curve would require a non-linear kernel (RBF/polynomial).
  3. B is roughly equidistant from both classes' nearest points — maximum margin. The support vectors are the closest points from each side.
🔗 Related Practice Questions
Sub-topic — support vectors: What are they? Answer: training points exactly on the margin boundary. They alone determine w — other points could be removed without changing the model.
Sub-topic — soft margin C parameter: What does it control? Answer: penalty for misclassifications. High C = strict (small margin, no errors). Low C = lenient (wider margin, tolerates errors).
Sub-topic — kernel for curved boundary: If the true boundary IS curved, what to use? Answer: RBF kernel K(x,x′)=exp(−γ‖x−x′‖²) or polynomial kernel. Maps to higher-dim space where curve becomes linear.
Sub-topic — why max margin generalizes: Answer: VC theory — larger margin = lower bound on test error. Intuitively: more room before a new point gets misclassified.
V17
A 2D dataset has positives (+) in the center forming a small cluster, negatives (−) around them in a ring. Will logistic regression do well? If yes, why? If no, what adaptation will make it work?
⌄
Inner +, outer ring of − · cannot be separated by any straight line
✅ Official Answer
Logistic regression will NOT do well — it is a linear method, and this dataset requires a circular separator. It can be adapted by using squares of the features (F₁² and F₂²) as additional features. In the expanded space, the boundary becomes linear (a hyperplane in 4D).
📐 The Math Behind The Fix
  1. A circle: F₁² + F₂² = r². Rearranged: F₁² + F₂² − r² = 0.
  2. Define new features: z₁ = F₁², z₂ = F₂². The boundary becomes: z₁ + z₂ = r². Linear in z!
  3. Logistic regression on (z₁, z₂) finds this linear boundary easily.
  4. This is the kernel trick's intuition without using kernels explicitly — basis expansion.
🔗 Related Practice Questions
Sub-topic — alternative method: What other method handles this naturally? Answer: kernel SVM with RBF kernel, or any neural network with hidden layers, or decision tree (axis-aligned splits form a box approximation), or kNN.
Sub-topic — DBSCAN: Would K-Means cluster these correctly? Answer: No — K-Means makes spherical clusters by centroid distance, would split the ring in half. DBSCAN (density-based) handles arbitrary shapes.
Sub-topic — XOR dataset: What's the basis expansion that linearizes XOR? Answer: add x₁·x₂ as feature. XOR(x₁,x₂) = x₁ + x₂ − 2·x₁·x₂ is linear in {x₁, x₂, x₁x₂}.
V18
A method uses a representation that can represent ANY function from features to target. Will it be prone to underfitting, overfitting, or neither? Explain in one sentence.
⌄
✅ Official Answer
Overfitting. Its representation is very expressive and could easily fit any training data — including noise — leading to poor generalization on test data.
💡 Capacity vs Generalization Trade-off
High-capacity models (deep nets, full-depth decision trees, kNN with k=1, kernel SVM with high γ) can fit anything → high variance → overfit easily. Low-capacity (linear regression on raw features) → high bias → underfit. The sweet spot needs regularization OR more data.

Sub-topic — predict-the-majority: High bias or variance? Answer: high bias — same prediction regardless of input. Underfits hopelessly.
Sub-topic — preventing overfitting: How in decision trees? In kNN? Answer: Decision trees → pruning (or limit max_depth, min_samples_leaf). kNN → increase k (averages out noise).

🔍 Hierarchical Clustering (HAC)

Q1 from Sample-Final-Exam2
V19
2D examples A, B, C, D positioned roughly: A far left, B middle, C and D close together on the right. Show HAC iterations using single-link and Euclidean distance.
⌄
A, B, C, D on a horizontal line · C and D are closest (smallest gap)
✅ Official Answer
  • Iteration 1 (start): (A), (B), (C), (D) — every point its own cluster
  • Iteration 2: (A), (B), (C, D) — C and D are closest, merge first
  • Iteration 3: (A), (B, C, D) — single-link distance from {C,D} to B is min(d(B,C), d(B,D)) which is smaller than d(A, B)
  • Iteration 4: (A, B, C, D) — final merge
📐 Single-Link Distance Recap
  1. Single-link: d(cluster1, cluster2) = min over all pairs (xᵢ ∈ cluster1, xⱼ ∈ cluster2) of d(xᵢ, xⱼ).
  2. Iteration 2: smallest pairwise distance is d(C, D). Merge.
  3. Iteration 3: now compare (A, B), (A, {C,D}), (B, {C,D}). Single-link of (B, {C,D}) = min(d(B,C), d(B,D)) = d(B,C). Compared to d(A,B), d(B,C) is smaller. Merge B with {C,D}.
  4. Iteration 4: only A and {B,C,D} remain. Merge.
🔗 Related Practice Questions
Sub-topic — complete-link: Would the iterations change? Answer: possibly yes — complete-link uses MAX distance. d({C,D}, B) = max(d(C,B), d(D,B)) = d(D,B), still smaller than d(A,B). Same result here but could differ for other layouts.
Sub-topic — dendrogram: What is it? Answer: tree showing the merge sequence with heights = merge distances. Cut horizontally at any level to get any number of clusters.
Sub-topic — K-Means vs HAC: Why use HAC instead? Answer: don't need to pick K up front; dendrogram exposes structure at every scale. Trade-off: O(n³) vs K-Means's O(nKd).
Sub-topic — Ward linkage: What does it minimize? Answer: variance increase when merging — favors compact, spherical clusters (like K-Means but hierarchical).

Decision Tree. It will pick up the relevant feature in the first split (highest information gain) and ignore the 99 irrelevant ones.

  2. 1-NN: Euclidean distance includes all 100 features equally. The 99 irrelevant features dominate the distance — true neighbors get lost.

🔗 Related Practice Questions

V23
A 100-layer deep neural network with logistic activation barely changes weights during training. Increasing the learning rate doesn't help. What's the problem? What fixes it?
⌄
✅ Official Answer
The vanishing gradient problem — logistic (sigmoid) derivatives ≤ 0.25, multiplied through 100 layers during backprop, drive gradients to ~0 in early layers. Fix: use ReLU instead of sigmoid.
📐 Why Learning Rate Doesn't Help
The gradient is essentially zero, not just small. η · 0 = 0 for any η. You need to remove the source of the vanishing, not amplify zeros. Other fixes: residual connections (ResNet), batch normalization, careful Xavier/He initialization, gradient clipping (for explosion).
🔗 Related Practice Questions
Sub-topic — exploding gradient: What is it and how to fix? Answer: gradients grow exponentially (large W or unbounded activations) → NaN losses. Fix: gradient clipping, lower learning rate, careful init.
Sub-topic — ResNet skip connections: How do they help? Answer: identity shortcut: y = F(x) + x. Gradient flows through identity path unchanged regardless of depth — bypasses vanishing.
Sub-topic — why not just use linear activation everywhere? Answer: stacked linear layers collapse into one linear layer (W₁W₂x = Wx). No non-linearity → no expressiveness gain from depth.

What is the vanishing gradient problem? Why does it occur with sigmoid activations in deep networks? Name two solutions.
⌄
Answer
Sigmoid derivative: σ'(z) = σ(z)(1−σ(z)) ∈ (0, 0.25]. In a 10-layer network, gradients are multiplied 10 times — 0.25¹⁰ ≈ 0.000001. Early layers receive nearly zero gradient → they barely update → model fails to learn.
Plain English
Like a game of telephone — each person (layer) whispers a quieter message. By the time the correction signal reaches the first layers, it's basically silence. The front layers don't learn.
Two Solutions
1. ReLU activation — derivative is 1 for z>0, so no shrinking across layers.
2. Residual connections (ResNet) — skip connections provide a gradient highway that bypasses layers.

Vanishing Gradient in RNNs
Gradient at step t must be multiplied by Wₕ and tanh' at every step back. Over 100 steps, if |Wₕ·tanh'| < 1, gradient → 0. The early steps learn nothing from long-range context.
LSTM Solution: Gates
LSTM adds a cell state (Cₜ) — a conveyor belt that carries information unchanged, modified only by multiplicative gates:

Forget gate (f): σ(Wf·[hₜ₋₁, xₜ]) — what to erase from cell state.
Input gate (i): σ(Wi·[hₜ₋₁, xₜ]) — what new info to write.
Output gate (o): σ(Wo·[hₜ₋₁, xₜ]) — what to expose as hidden state.

Gradients flow through cell state with minimal multiplication → no vanishing.
Plain English
RNN has a leaky short-term memory — old memories fade. LSTM is like writing on a whiteboard with a dedicated eraser (forget gate) and a separate marker (input gate). Important facts written on the board persist, while irrelevant ones get erased.

Concentric Circles Dataset
Why K-Means Fails
K-Means partitions by Euclidean distance to centroids — it creates linear/convex boundaries. Both centroids end up in the middle, splitting by left/right or up/down, not inner/outer.
Solution 1: DBSCAN

Explain the bias-variance tradeoff. Write the decomposition of expected generalization error. What causes high bias? High variance?
⌄
Error Decomposition
Symbol Breakdown
TermMeaningCaused by
Bias²Systematic error — model too simple to capture patternsUnderfitting, simple model (linear for nonlinear data)
VarianceSensitivity to training data fluctuationsOverfitting, complex model (deep tree, k=1 KNN)
Noise (σ²)Irreducible error — inherent randomness in dataMeasurement error, label noise — can't be eliminated
Plain English
Bias = your model has wrong assumptions (a ruler trying to fit a curve). Variance = your model is too sensitive to exactly which data you saw (memorized noise). Reducing one tends to increase the other. Goal: find the sweet spot using cross-validation.
Fixes
High bias → more complex model, more features, less regularization.
High variance → more data, dropout, regularization (L1/L2), simpler model, ensembles.

SVM Optimization Problem
Symbol Breakdown
SymbolMeaning

‖w‖Norm of w — margin = 2/‖w‖ so minimizing ‖w‖ maximizes margin

Kernel Trick
Instead of computing φ(x) explicitly in high-dim space, use K(xᵢ,xⱼ) = φ(xᵢ)·φ(xⱼ). RBF kernel: K = exp(−‖xᵢ−xⱼ‖²/2σ²) maps to infinite dimensions — makes any distribution linearly separable.
SVM Margin Visualizer
Plain English
SVM draws the widest possible street between two classes. The houses (points) right on the edge of the street are the support vectors. The kernel trick is like inflating a flat map into a globe — what looked like a tangle becomes separable in 3D.

Compare L1 (Lasso) and L2 (Ridge) regularization. Write their loss functions. When does L1 produce sparse weights?
⌄
Regularized Loss Functions
Symbol Breakdown
SymbolMeaning
LOriginal loss (MSE, cross-entropy, etc.)
λRegularization strength — higher = more penalty on large weights
Σ|wⱼ|L1 penalty — sum of absolute values. Geometry: diamond shape → corners at axes → sparsity
Σwⱼ²L2 penalty — sum of squares. Geometry: sphere → touches axes at smooth curve → no sparsity
Why L1 Creates Sparsity
The L1 constraint region (diamond) has corners exactly on the axes. The loss function's minimum is most likely to touch a corner where one or more weights = 0 exactly. L2 (sphere) is smooth — minimum touches at a point with small but non-zero weights.
Plain English
L2 shrinks all weights toward zero proportionally (everyone gets a pay cut). L1 can cut some weights to exactly zero (fire some workers entirely). Use L1 when you suspect many features are irrelevant — it automatically does feature selection.

What is PCA? Given a 2D dataset, describe the steps to compute the first principal component. What does the explained variance ratio mean?
⌄
PCA Objective
Steps
  1. Center the data: subtract the mean from each feature
  2. Compute the covariance matrix: C = (1/n)XᵀX
  3. Compute eigenvalues (λ) and eigenvectors (v) of C
  4. Sort by eigenvalue descending — largest λ = most variance explained
  5. Project data onto top-k eigenvectors: Z = XV_k
Explained Variance Ratio
EVR for component i = λᵢ / Σλⱼ. If PC1 has EVR=0.85, it captures 85% of total variance. Choose k components to retain ≥95% of variance.
Plain English
PCA finds the directions (axes) along which the data spreads out most. The first axis captures the most variation. Think of projecting a 3D point cloud onto a 2D plane oriented for maximum spread — you lose minimal information.
When NOT to Use PCA
❌ When features are not correlated (PCA won't help much)
❌ When interpretability matters (principal components mix features)
❌ Before tree-based models (they handle high dimensions well anyway)

What is cross-entropy loss? When is it preferred over MSE for classification? Compute cross-entropy for ŷ=[0.7, 0.2, 0.1], true class=0.
⌄
Cross-Entropy Loss
Symbol Breakdown
SymbolMeaning
yᵢTrue label (one-hot: 1 for correct class, 0 otherwise)
ŷᵢPredicted probability for class i
logNatural log — penalizes very wrong confident predictions severely
−ΣNegative sum (log of probability < 1 is negative, so we negate)
Calculation
True class = 0, so y = [1, 0, 0]
Loss = −(1·log(0.7) + 0·log(0.2) + 0·log(0.1)) = −log(0.7) ≈ 0.357
Why Not MSE for Classification?
MSE treats all errors equally. A prediction of 0.49 vs 0.51 (wrong class) gets same weight as 0.01 vs 0.99. Cross-entropy penalizes confident wrong predictions exponentially — much stronger gradient signal for learning.

What is batch normalization? Write the formula. Why does it help training? When should you NOT use it?
⌄
Batch Norm Formula
Symbol Breakdown
SymbolMeaning

Plain English
Each layer's inputs drift as weights change — BN resets them to a stable distribution per batch, like recalibrating a scale before each use. This lets you use higher learning rates and makes training less sensitive to initialization.
When NOT to Use BN
❌ Very small batches (batch size 1-4) — statistics are unreliable. Use Layer Norm instead.
❌ RNNs — sequence lengths vary; use Layer Norm.
❌ When you need consistent behavior regardless of batch composition.

Explain dropout regularization. If a neuron has output 0.8 with dropout rate p=0.5 during training, what is the expected output at test time?
⌄
Answer
During training: each neuron independently zeroed out with probability p=0.5. Remaining neurons scaled by 1/(1−p) = 2 (inverted dropout) to keep expected value the same.

Training output (if not dropped): 0.8 × 2 = 1.6
Expected: 0.5 × 1.6 + 0.5 × 0 = 0.8

At test time: use all neurons, no scaling needed (already handled by training scaling).
Plain English
Dropout is like studying with random teammates missing — you can't rely on any one teammate, so you learn to do the job yourself. Each neuron learns robust features independently. At test time, the whole team shows up but each member contributes at half intensity (or equivalently, training scaled up so test needs no change).
